Pulled from EPIC: `.agent/plan/epics/007.2-migrate-cli-to-commander-js.md`.

Verification gate (binding, from the EPIC's `## Verification Gate` section):

> Gates: `npm run verify` (typecheck + test + verify:handoff + lint + db status).
> Hermetic tests use a local `Command`, `.exitOverride()`, and configured output so
> Commander.js never terminates the test process. They prove the complete audited
> command inventory is registered exactly once; each leaf has a description,
> documented arguments/options, and example help; async actions run through
> `.parseAsync()`; required, repeatable, positional, boolean, and choice-like inputs
> reach existing handlers with the same values; unknown commands/options, missing
> values, and excess arguments fail on stderr with non-zero status; handler
> stdout/stderr and exit codes are preserved; and the five old spellings are
> rejected. Existing handler and end-to-end tests remain green. An architecture
> test proves `src/apps/cli/index.ts` only builds the tree and that every leaf
> command is registered from a separate command file.
>
> Proof: run the real program against a temporary SQLite database, assert generated
> top-level/group/leaf help, reject an old route, execute a real create/list
> workflow through Commander.js, and exercise the highest-risk migrations — the two
> renamed output routes (`list model`, `list event`) and a positional argument
> (`export initiative <id>`).
>
> ```bash
> set -euo pipefail
> export KANTHORD_DB="$(mktemp -d)/kanthord.db"
>
> node src/main.ts --help > /tmp/kanthord-help.txt
> grep -q 'Usage: kanthord' /tmp/kanthord-help.txt
> grep -q 'create' /tmp/kanthord-help.txt
> grep -q 'db' /tmp/kanthord-help.txt
>
> node src/main.ts create project --help > /tmp/create-project-help.txt
> grep -q 'Usage: kanthord create project' /tmp/create-project-help.txt
> grep -qi 'create' /tmp/create-project-help.txt
> grep -q -- '--name <name>' /tmp/create-project-help.txt
> grep -qi 'example' /tmp/create-project-help.txt
>
> node src/main.ts db migrate --help > /tmp/db-migrate-help.txt
> grep -q 'Usage: kanthord db migrate' /tmp/db-migrate-help.txt
>
> if node src/main.ts daemon run --help >/tmp/old-route.out 2>/tmp/old-route.err; then
>   echo 'FAIL: old daemon run route must be removed'; exit 1
> fi
> grep -qi 'unknown command' /tmp/old-route.err
>
> node src/main.ts db migrate
> PROJECT=$(node src/main.ts create project --name commander-proof)
> INITIATIVE=$(node src/main.ts create initiative --project "$PROJECT" --name cli)
> OBJECTIVE=$(node src/main.ts create objective --initiative "$INITIATIVE" --name routing)
> TASK=$(node src/main.ts create task --objective "$OBJECTIVE" \
>   --title 'migrate parser' --instructions 'Route the CLI with Commander.js.' \
>   --ac 'Commander.js parses every command')
>
> test "$(node src/main.ts find project --name commander-proof)" = "$PROJECT"
> node src/main.ts list task --initiative "$INITIATIVE" --json > /tmp/tasks.json
> node -e 'const rows=require("/tmp/tasks.json");process.exit(rows.length===1&&rows[0].id===process.argv[1]?0:1)' "$TASK"
> node src/main.ts run daemon --help | grep -qi 'run'
>
> # renamed output route (get models → list model): wires end to end, real catalog
> node src/main.ts list model --json > /tmp/models.json
> node -e 'process.exit(Array.isArray(require("/tmp/models.json"))?0:1)'
>
> # renamed route over real SQLite (events → list event): queries events table, exits 0
> node src/main.ts list event --after 0 --json
>
> # positional-argument promotion (export initiative <id>, not --id)
> EXPORT_DIR="$(mktemp -d)"
> node src/main.ts export initiative "$INITIATIVE" --out "$EXPORT_DIR"
> test -f "$EXPORT_DIR/cli/INDEX.md"
> echo 'PROOF OK'
> ```
