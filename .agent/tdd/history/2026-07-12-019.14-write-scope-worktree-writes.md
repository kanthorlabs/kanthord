### Security containment confirmation

All three properties cited in the dispatch hold:

1. **Paths outside the worktree still blocked.** Role policy (layer 1) uses `allow: [wtRoot, wtRoot + "/**"]`; any absolute path outside that range is denied before the write-scope check is reached. The `"**"` sentinel in `isPathInScope` is irrelevant for such paths.

2. **Sentinel only authorises whole-repo when `write_scope` explicitly declares `**`.** `isPathInScope` checks `ns === "**"` only after `normalizeScopePath` runs; a scope entry like `"src/**"` normalizes to `"src"` (not `"**"`), so the sentinel is not triggered.

3. **Allowlist widening bounded to worktree root and subtree.** `wtAllow = [wtRoot, wtRoot + "/**"]` covers exactly the bare root (exact regex `^<wt>$`) and subtree (`^<wt>/.*$`). No wildcards introduce unintended breadth.
