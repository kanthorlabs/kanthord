## TEST-ENGINEER — S2 list-event-truncation · RED: truncation sentinel missing

**Cycle.** RED for Story `S2` (`src/apps/cli/events.test.ts`).

**Handoff verification.** `npm run verify:handoff` → `VERIFY: PASS` (exit 0); S1 tests confirmed GREEN (11 pass, 0 fail in `list-tasks.test.ts` + `list-tasks.test.ts`).

**Test written.**

- file: `src/apps/cli/events.test.ts` (edited) — known rewrite per Story 02 "Known test change": old test "events --limit 2 makes three immediate reads for 5 events…" replaced by four new S2 tests:
  - `S2: non-follow --limit 2 --json over 5 events emits 2 ndjson lines then nextCursor sentinel`
  - `S2: non-follow --limit 2 (human) over 5 events emits 2 lines then more-available hint`
  - `S2: non-follow --limit 10 --json covers all 5 events emits no sentinel`
  - `S2: non-follow --after last event id emits nothing`
- asserts: a non-follow `--limit 2` over 5 events stops at the first page (2 events), performs a lookahead, then emits `{"nextCursor":"B"}` as a trailing stdout line (JSON mode) or `more available — pass --after B` as a trailing stderr line (human mode); when the page covers the tail or the cursor is exhausted, no sentinel/hint is emitted.

**RED proof.**

- command: `node --test src/apps/cli/events.test.ts`
- exit: 1 — 2 failing, 9 passing
- failing assertion 1: `AssertionError: 2 event lines + 1 sentinel on stdout — 5 !== 3` (current code auto-drains all 5 events, no sentinel)
- failing assertion 2: `AssertionError: 2 event lines + 1 hint on stderr — 5 !== 3` (same cause)
- The "no-sentinel" and "empty" cases pass trivially — current code never emits a sentinel, which is already correct for those paths; sensitivity is demonstrated by the two failing truncated-page tests.

**Open to Software Engineer.**

- Seam: `runEvents` in `src/apps/cli/events.ts` (non-follow branch only).
  - Remove the non-follow auto-drain: a non-follow read exits after ONE page regardless of `page.length === limit`.
  - After emitting the page events, perform a lookahead — call `listEvents.execute({ after: lastCursor, limit: 1 })` — to detect whether more events exist.
  - If the lookahead returns ≥ 1 event and `json` is true: push `JSON.stringify({ nextCursor: lastCursor })` to `stdout`.
  - If the lookahead returns ≥ 1 event and `json` is false: push `more available — pass --after ${lastCursor}` to `stderr`.
  - If the lookahead returns 0 events (or the page was empty): emit nothing.
  - `--follow` branch: unchanged — keep streaming with poll/sleep loop, no sentinel.
  - No port, use-case, or domain change needed.
