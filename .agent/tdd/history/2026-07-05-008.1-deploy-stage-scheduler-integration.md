# Implementation cycle — 008.1-deploy-stage-scheduler-integration

Pulled from EPIC: `.agent/plan/epics/008.1-deploy-stage-scheduler-integration.md`.

Verification gate (binding, from the EPIC's `## Verification Gate` section):
> - `npm run typecheck` exits 0; `npm test` green for all Story suites.
> - **Closes B2:** a compiled plan's deploy-stage nodes are dispatched by the scheduler continuing past PR-open — asserted against the **real `pollOnce` dispatch lifecycle** (a deploy-stage node enters the dispatchable set, is leased, goes `pending`->`running`, and is executed through the scheduler), not a bare `runChain`/`runStage` call.
> - **Uniform gating, no structural-node aggregate:** the first deploy stage is gated by task predecessors (compiler-emitted terminal-task->deploy edges) and each later stage by its predecessor deploy stage, under one generalized predecessor-gate rule; a deploy stage is not ready before all predecessors have `exit_gate_passed=1`.
> - **Regression:** existing task-node dispatch, gating, leases, and generation behavior are unchanged (a deploy-free plan schedules identically).
> - **Generation isolation:** deploy-stage dispatch respects the same generation/dirty-plan guards as tasks — a stale-generation deploy node is not dispatched, and its dispatched generation is pinned.
> - **Pass advances / fail blocks:** a passing stage marks its exit gate and only then does the next deploy stage become dispatchable; a failing stage resolves `halt_and_escalate` with evidence, does not pass its gate, and downstream deploy nodes are never dispatched or executed. A passed stage and a running stage are never re-executed within a poll cycle.
> - **Completion:** the last deploy stage's passed exit gate is the observable completion of the deploy chain — no further deploy node dispatches after it.
> - **No auto-merge (non-vacuous):** across the whole run the fake broker's recorded command log shows no merge/deploy/rollback verb, and a passing stage still emits a `notify_human` event with the stage context (asserted against the recorded log, not a vacuous absence) (PRD §7.4, §9).
