> Gates: `npm run verify` (typecheck + test + verify:handoff + lint + db status).
> Hermetic — no network. The suite proves each fix with fakes / real-git-in-temp.
> F1: LocalWorkspaceManager emits an absolute candidate workspace even from a RELATIVE root; GitRepositoryLanding.land throws a typed invariant error on a relative workspace path; a landing test runs the real adapter with a relative root+DB in a temp cwd and lands successfully.
> F2: a migration test asserts (a) every domain EVENT_TYPES member inserts into the migrated events table, (b) an unknown/misspelled type is REJECTED, (c) pre-existing event rows and indexes survive the rebuild. Explicit column lists; FK handling decided before BEGIN.
> F3: ApproveTask returns a discriminated outcome — fast-forward → approved; LandingConflictError → conflict (task still awaiting_confirmation, task.conflict event recorded, no throw); generic error → landing_failed carrying safe user text + an internal cause. feed.append failure while recording a conflict MUST NOT report approved. CLI action test: each outcome maps to the right exit code + a single sanitized line.
> Proof: deterministic — pure CLI + real git in temp dirs, NO model — proving all three outcomes on a RELATIVE DB path.
