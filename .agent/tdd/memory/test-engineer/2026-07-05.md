## 2026-07-05 — Epic 010 T1 guard root cause

- Node.js 24 `net.createConnection` and `http.request` call `Socket.prototype.connect` with normalized args as `args[0]` (array-like `{0: options, 1: listener}`), NOT a plain options object; `args[0].host` is `undefined` so the guard's `netHost` returns null.
- `tls.connect`/`https.request`/`http2.connect` pass a plain options object (`args[0].host` is the hostname string) — different code path.
- Fix: in `netHost`, add `0 in first` check before the plain-object branch; extract `first[0]` as the inner options object.
- Empirically verified: the fixed `netHost` blocks `net.createConnection(9999,"8.8.8.8")`, `http.request("http://example.com")`, and allows `net.createConnection(9999,"127.0.0.1")` loopback.
