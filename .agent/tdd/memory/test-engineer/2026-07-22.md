  RED cycle (same pattern as 007.8's S1/S2).
- `access()`/`fs.access` on a directory only needs search permission on its
  _parent_, not on itself — `chmod 000` on the target dir still passes
  `pathExists`, but a child process `execFile(..., { cwd })` into it fails with
  `spawn git EACCES`. Good way to force a "git-error" distinct from "not a repo".

  "Failed to clone" error. `lstat` is required to classify it correctly _before_
  any clone attempt — asserted this via "message must not say 'Failed to clone'".
- `git rev-parse --git-dir`/`--show-toplevel` from an empty dir nested inside
  another git worktree resolves to the _outer_ repo — confirmed via a guard
