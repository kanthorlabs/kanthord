## 2026-07-15 — Story 006 T1+T2 (Budgets & Daemon Ops) — direct dispatch

- **DaemonClient type change**: `triggerVerify` redeclared in `lib/client.ts` as a structural method with `Promise<{report?: VerifyReportVM}>` return type instead of the proto-es `Message<T>` type. The test file uses `DaemonClient["triggerVerify"]` (not `vi.fn()`) for fake lambdas; proto-es v2 requires `$typeName` on all Message shapes, so plain object lambdas caused TS2322. The fix: `DaemonClient = Omit<Client<DaemonService>, "triggerVerify"> & { triggerVerify(...): Promise<{report?: VerifyReportVM}> }` — structurally compatible with both real client (cast via `as unknown as DaemonClient`) and plain object fakes.
- **ConfirmActionDialog**: changed `onConfirm: () => void` → `onConfirm: (inputValue: string) => void`; call site changed from `onClick={onConfirm}` to `onClick={() => onConfirm(inputValue)}`. TypeScript allows fewer-param callbacks in calling position (existing callers `handleHalt: () => void` remain valid). This lets Budgets capture the typed reason from the dialog's managed input.
- **BreakerStateBadge**: closed→outline (success tone), open→destructive (danger), half-open→outline (warning), unknown→secondary (neutral fallback) — exact pattern from FeatureStatusBadge.
- **OpsPage**: minimal card-grid template; `data-testid={locators.opsPage.root}` on the grid container; children rendered inside.
- **DaemonOps**: `getDaemonStatus({})` on mount; loading → DataStates; health card conditioned on `lastPing?.present === true`; tasksProcessed renders `String(bigint)` (distinct from unavailable); verify trigger calls `triggerVerify({})` and shows report JSON inline.
