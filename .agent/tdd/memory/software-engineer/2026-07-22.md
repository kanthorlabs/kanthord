## 2026-07-22 — Story 01 checkout-root inspection (007.9)

- Replaced boolean `isGitRepo()` with `inspectCheckout()` returning a 5-state
  discriminated union (root-checkout/enclosing-checkout/bare/not-a-repo/
  git-error), built on two `git rev-parse` calls (`--is-bare-repository` then
  `--show-toplevel` + `realpath` compare) so an enclosing worktree is never
  misread as "home exists".
- Key design call: emptiness (readdir === 0 entries) takes priority over
  enclosing-checkout/not-a-repo classification for the clone-fresh decision —
  Proof A's nested-but-empty dir must still clone; only a non-empty
  enclosing-checkout/not-a-repo dir is a hard error. root-checkout and
  git-error/bare never fall through to clone.
- Added `inspectPath()` (lstat-based) ahead of `inspectCheckout` so a broken
  symlink is classified before any clone attempt, never misread as absent.
- Verified empirically (outside the test runner) that (a) spawning git into a
  chmod-000 cwd rejects with `EACCES` in the message, and (b) POSIX `rename`
  onto an already-existing empty dir succeeds — both load-bearing for the
  GREEN implementation and both confirmed rather than assumed.

## 2026-07-22 — Story 02 provider transient-retry (007.9)

- Retry loop lives entirely in `RunNextTask` (between tx1/tx2), gated on
  `{failed, transient:true}`; bounded by an injectable `maxAttempts` (default
  3, per the investigation note that the SDK already retries some HTTP noise
  below the turn boundary) and a hardcoded 120s elapsed cap. Backoff is
  exponential-with-full-jitter capped at 5s, floored by `retryAfterMs`.
- `pi.ts`'s Codex error classifier (Contract item 2) has no driving test this
  cycle — left untouched rather than speculatively implemented; the retry
  loop is fully wired to pick it up once `pi.ts` starts setting `transient`.
- Real design fight this turn: `daemon.test.ts`'s pre-existing T3a fake (2
  params, 2nd=logger) and the new S2 capturing fake (2 params, 2nd=
  failTransient) put two different things in the same argument slot across
  two test files I can't edit. Fixed positional args can't satisfy both
  (JS drops extra args, doesn't "skip" a middle one). Resolved via
  `buildDaemon.length` (real arity) dispatch: the real 3-param composition
  factory always gets all 3 args (no logger regression); a 2-param fake picks
  failTransient vs logger for slot 2 based on whether --fail-transient was
  actually given.
- Added `"provider.retry"` to `EVENT_TYPES` — required (not optional) because
  the test files themselves compare `e.type === "provider.retry"`, which
  fails `tsc --noEmit` without the real literal; this will break
  `event.test.ts`'s hardcoded 16-literal count assertion, flagged to the TE
  as a one-line follow-up (test-file lane, not mine).
