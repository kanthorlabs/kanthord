### 2026-07-17 — B2 upsert fix

- Changed three plain `INSERT` statements in `sqlite-project-repository.ts` and `sqlite-initiative-repository.ts` to `INSERT … ON CONFLICT(id) DO UPDATE SET name = excluded.name` upserts.
- Motivation: rename-crash regression tests called `save`/`saveObjective` with the same id and a new name, which hit a UNIQUE constraint with a plain INSERT.
- Only `name` is updated on conflict; `id` and foreign-key columns (`projectId`, `initiativeId`) are immutable and not touched by the DO UPDATE clause.
