# EPIC 007.7 — dependency & event-feed observability

> **Authored 2026-07-21** from the post-007.5 TODO-API E2E (see
> `007.6-guided-conflict-resolution.md` Appendix B). Batches the two minor
> observability findings that block "pull progress" from being trustworthy on the
> CLI: the DAG is invisible from `list task`, and `list event` silently truncates
> at its default page. Both are small, program-level testable CLI fixes.
>
> **Grounded (verified 2026-07-21 against the live CLI):**
>
> - `list task --initiative <id> --json` returns per task
>   `{ id, title, status, state, waiting }` — it shows the _runtime_ `waiting`
>   set but **omits the static `dependencies` edges** (BUG-4). The DAG the graph
>   declared is not observable from the list. The domain entity already carries
>   `dependencies: string[]` (`src/domain/task.ts:19`), so this is a projection
>   gap, not a model gap.
> - `list event --after 0 --json` capped at exactly 100 rows on a run that had
>   more; re-querying with `--limit 1000` returned the full set. There is a
>   `--limit` flag (default page ~100) but **no signal that the page was
>   truncated** — no next-cursor, no "N more" — so a reader who does not know to
>   raise `--limit` or page by cursor silently sees a partial, stale feed and can
>   mis-conclude that terminal events never fired (N5). This nearly produced a
>   false "silent completion" data-loss report during the E2E.

## Goal

After 007.7, the CLI makes the task DAG and the full event history observable
without guesswork. `list task --json` includes each task's declared
`dependencies` (the static edges) alongside the existing runtime `waiting` set,
so a client can reconstruct the graph from one call. `list event` **signals when
a page is truncated**: when more events exist beyond the returned page, the CLI
emits the next cursor (and, in human mode, a "more available — pass
`--after <cursor>`" line), so no reader mistakes a capped page for the whole
feed. No schema changes, no new tables — `dependencies` already exists on the
entity; the event cursor (ULID) already sorts by time.

## Verification Gate

Gates: `npm run verify` (typecheck + test + verify:handoff + lint + db status).
Hermetic — no network. Fakes + real-git-in-temp where needed:

- **S1 (deps visible):** a `list task --json` action/unit test — for an imported
  graph with edges (siblings depend on the root), each task's JSON includes a
  `dependencies` array equal to its declared edges (root → `[]`; each sibling →
  `[<root-id>]`), in addition to the existing `waiting`. The two are distinct:
  `dependencies` is static, `waiting` is the currently-unsatisfied subset.
- **S2 (truncation signalled):** a `list event` action test — with N events and a
  page `--limit` < N, the JSON output includes a resumable next cursor (the last
  returned event's id) and the human output prints a "more available" hint naming
  `--after <cursor>`; when the page covers the tail (nothing beyond it), no hint
  is printed. `--follow` already streams past the cursor and must be unaffected.

Proof: deterministic — pure CLI, NO model — import the TODO graph (declares
`depends-on` edges) on a **relative** DB, seed events via the fake-agent daemon,
and assert both the dependency projection and the truncation signal.
`set -euo pipefail`; expected failures use `if ! …; then`.

## Stories

- **S1 — Project `dependencies` in `list task --json`.** Add the task's declared
  `dependencies` (from the entity) to the list read model beside `waiting`.
  Query-side only (CQRS-lite read), no domain change. Human output may show a
  compact deps count/ids; JSON carries the full array.
- **S2 — Truncation signal for `list event`.** When the returned page does not
  reach the tail, include a resumable next cursor in `--json` (e.g. `nextCursor`)
  and a "more available — `--after <cursor>`" hint in human output; when the page
  covers the tail, emit neither. Leave `--follow` semantics unchanged.

## Non-goals

- **BUG-6 — rename `dependsOn`/`depends-on` → `dependencies` everywhere** (128
  remaining sites in the import/graph/codec layers + the `depends-on:` YAML key
  used under `scripts/`, NO back-compat — Ulrich 2026-07-21). Because it edits
  lane-forbidden config/scripts and the import contract, it is a **maintainer
  epic** (executed directly by human + assistant, not dispatched through
  `/work`), tracked separately. 007.7 only _projects_ the already-named
  `dependencies` entity field; it does not touch the import key.
- **New event types, richer payloads, or a push/stream transport** — out of
  scope; the pull cursor feed is unchanged apart from the truncation signal.
- **Pagination redesign** (offset paging, total counts) — the ULID cursor stays
  the paging primitive; we only surface it when a page is capped.
