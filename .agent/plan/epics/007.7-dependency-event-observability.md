# EPIC 007.7 — dependency & event-feed observability

> **Authored 2026-07-21** from the post-007.5 TODO-API E2E (see
> `007.6-guided-conflict-resolution.md` Appendix B). Batches the two minor
> observability findings that block "pull progress" from being trustworthy on the
> CLI: the DAG is invisible from `list task`, and `list event` silently truncates
> at its default page. Both are small, program-level testable CLI fixes.
>
> **Grounded (verified 2026-07-21 against the live CLI):**
>
> - `list task --initiative <id> --json` returns per task
>   `{ id, title, status, state, waiting }` — it shows the _runtime_ `waiting`
>   set but **omits the static `dependencies` edges** (BUG-4). The DAG the graph
>   declared is not observable from the list. The domain entity already carries
>   `dependencies: string[]` (`src/domain/task.ts:19`), so this is a projection
>   gap, not a model gap.
> - `list event --after 0 --json` capped at exactly 100 rows on a run that had
>   more; re-querying with `--limit 1000` returned the full set. There is a
>   `--limit` flag (default page ~100) but **no signal that the page was
>   truncated** — no next-cursor, no "N more" — so a reader who does not know to
>   raise `--limit` or page by cursor silently sees a partial, stale feed and can
>   mis-conclude that terminal events never fired (N5). This nearly produced a
>   false "silent completion" data-loss report during the E2E.

## Goal

After 007.7, the CLI makes the task DAG and the full event history observable
without guesswork. `list task --json` includes each task's declared
`dependencies` (the static edges) alongside the existing runtime `waiting` set,
so a client can reconstruct the graph from one call. `list event` **signals when
a page is truncated**: when more events exist beyond the returned page, the CLI
emits the next cursor (and, in human mode, a "more available — pass
`--after <cursor>`" line), so no reader mistakes a capped page for the whole
feed. No schema changes, no new tables — `dependencies` already exists on the
entity; the event cursor (ULID) already sorts by time.

## Verification Gate

Gates: `npm run verify` (typecheck + test + verify:handoff + lint + db status).
Hermetic — no network. Fakes + real-git-in-temp where needed:

- **S1 (deps visible):** a `list task --json` action/unit test — for an imported
  graph with edges (siblings depend on the root), each task's JSON includes a
  `dependencies` array equal to its declared edges (root → `[]`; each sibling →
  `[<root-id>]`), in addition to the existing `waiting`. The two are distinct:
  `dependencies` is static, `waiting` is the currently-unsatisfied subset.
- **S2 (truncation signalled):** a `list event` action test — with N events and a
  page `--limit` < N, the non-follow JSON output stays ndjson (one event per
  line) and appends a trailing sentinel line `{"nextCursor":"<lastId>"}`; the
  human output prints a "more available" hint naming `--after <cursor>`; when the
  page covers the tail (nothing beyond it), neither the sentinel nor the hint is
  printed. Non-follow returns a single page (no cross-page auto-drain). `--follow`
  already streams past the cursor, emits no sentinel, and must be unaffected.

Proof: deterministic — pure CLI, NO model — import the TODO graph (declares
`dependencies` edges) on a **relative** DB, seed events via the fake-agent daemon,
and assert both the dependency projection and the truncation signal.
`set -euo pipefail`; expected failures use `if ! …; then`.

```bash
set -euo pipefail
WORK="$(mktemp -d)"; cd "$WORK"
export KANTHORD_DB=".data/kanthord.db"
REPO_ROOT="$OLDPWD"
node "$REPO_ROOT/src/main.ts" db migrate >/dev/null
PROJECT=$(node "$REPO_ROOT/src/main.ts" create project --name obs | head -1)

HOME_REMOTE="$WORK/home.git"; git init -q --bare -b main "$HOME_REMOTE"
SEED="$WORK/seed"; git clone -q "$HOME_REMOTE" "$SEED"
git -C "$SEED" -c user.email=a@b.c -c user.name=t commit -q --allow-empty -m init
git -C "$SEED" push -q origin main
MIRROR="$WORK/mirror"
REPO=$(node "$REPO_ROOT/src/main.ts" create repository --project "$PROJECT" --name home \
        --remote-url "file://$HOME_REMOTE" --branch main --auth ambient --path "$MIRROR" | head -1)
CREDVAL="$(mktemp)"; printf 'dummy' > "$CREDVAL"
CRED=$(node "$REPO_ROOT/src/main.ts" create credential --project "$PROJECT" --name c1 --provider openai-codex --value-file "$CREDVAL" | head -1)
PROV=$(node "$REPO_ROOT/src/main.ts" create ai-provider --project "$PROJECT" --name p1 --provider openai-codex --model gpt-5.6-terra | head -1)

# make-landing-graph.sh is a superset of make-todo-graph.sh: it authors the same
# TODO-API graph (root "Create Task"; siblings like "List Tasks" depend on it)
# AND ships the .fake-agent.json the no-model daemon needs to emit events (B1).
GRAPH="$WORK/g"; "$REPO_ROOT/scripts/e2e/make-landing-graph.sh" "$GRAPH" >/dev/null
node "$REPO_ROOT/src/main.ts" import graph "$GRAPH" --create --project "$PROJECT" \
        --bind source="$REPO" --bind provider="$PROV" --bind cred="$CRED" >/dev/null
INIT=$(node -e 'console.log(JSON.parse(require("fs").readFileSync(process.argv[1]+"/.kanthord-export.json","utf8")).initiativeId)' "$GRAPH")

# --- BUG-4: list task --json exposes the declared dependencies edges ---
# Capture to a file first (drain-safe: an early-closing consumer pipe would give
# the producing CLI SIGPIPE under `set -o pipefail` — B2).
node "$REPO_ROOT/src/main.ts" list task --initiative "$INIT" --json > "$WORK/tasks.json"
node -e 'const ts=JSON.parse(require("fs").readFileSync(process.argv[1],"utf8"));
    const root=ts.find(t=>/Create Task/.test(t.title));
    const sib=ts.find(t=>/List Tasks/.test(t.title));
    if(!Array.isArray(root.dependencies)||root.dependencies.length!==0){console.error("root deps wrong");process.exit(1)}
    if(!sib.dependencies||!sib.dependencies.includes(root.id)){console.error("sibling dep missing");process.exit(1)}
    console.log("deps OK")' "$WORK/tasks.json"

# --- N5: list event signals truncation with a resumable cursor ---
export KANTHORD_FAKE_AGENT="$GRAPH/.fake-agent.json"
node "$REPO_ROOT/src/main.ts" run daemon --until-idle --poll-interval 200 >/dev/null
# a tiny page must advertise a next cursor (more events exist beyond it).
# The --json feed stays ndjson (one event per line); a truncated page appends a
# trailing sentinel line {"nextCursor":"<lastId>"}. Capture, then read that LAST line.
node "$REPO_ROOT/src/main.ts" list event --after 0 --limit 2 --json > "$WORK/page.ndjson"
CURSOR=$(node -e 'const L=require("fs").readFileSync(process.argv[1],"utf8").trim().split("\n").filter(Boolean);const last=JSON.parse(L[L.length-1]);if(!last.nextCursor){console.error("no next cursor on a truncated page");process.exit(1)}process.stdout.write(last.nextCursor)' "$WORK/page.ndjson")
test -n "$CURSOR"
# paging from the advertised cursor returns the remaining events (drain-safe: to a
# file, not `grep -q` on a live pipe, which would SIGPIPE the producer — B2).
node "$REPO_ROOT/src/main.ts" list event --after "$CURSOR" --limit 1000 --json > "$WORK/rest.ndjson"
grep -q '"type"' "$WORK/rest.ndjson"

echo "007.7 PROOF OK"
```

## Stories

- **S1 — Project `dependencies` in `list task --json`.** Add the task's declared
  `dependencies` (from the entity) to the list read model beside `waiting`.
  Query-side only (CQRS-lite read), no domain change. Human output may show a
  compact deps count/ids; JSON carries the full array.
- **S2 — Truncation signal for `list event`.** When the returned page does not
  reach the tail, include a resumable next cursor in `--json` (e.g. `nextCursor`)
  and a "more available — `--after <cursor>`" hint in human output; when the page
  covers the tail, emit neither. Leave `--follow` semantics unchanged.

## Non-goals

- **BUG-6 — rename `dependsOn`/`depends-on` → `dependencies` everywhere**
  (**DONE 2026-07-21**, maintainer epic executed directly by human + assistant).
  Plural list sites → `dependencies` (`PkgTask` field, YAML key, codec,
  `format.ts`, graph use-cases, `create task --dependencies`); singular edge
  sites → `dependencyId` / `--dependency` (`add`/`remove dependency`, port,
  SQLite adapter). Covered `src/`, `scripts/e2e/make-todo-graph.sh`, and
  `docs/formats/graph-md.md`; historical `.agent/**` + `docs/flowchart/**`
  records left as-is. NO back-compat. `npm run verify` green + real import proof.
- **New event types, richer payloads, or a push/stream transport** — out of
  scope; the pull cursor feed is unchanged apart from the truncation signal.
- **Pagination redesign** (offset paging, total counts) — the ULID cursor stays
  the paging primitive; we only surface it when a page is capped.
