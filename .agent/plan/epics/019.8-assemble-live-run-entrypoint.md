# 019.8 Assemble the Live Run Entrypoint

## Outcome

`kanthord run` becomes a working daemon end-to-end: it **clones the slot repo
itself** into a PAT-authenticated local checkout, runs each task in its own
worktree/branch, counts real commits ahead of base, and threads the Epic 019.7
broker delivery adapters into `runDaemon` — so a completed session's commits
actually become a real PR through the broker. Today every piece exists but the
entrypoint never assembles them: `run.ts` calls the *sync* `buildRealDeps` (no
identity/registry), `slot.repo` is a URL but `run.ts` treats it as a local path,
the `git.clone` adapter and `dispatchWorktree` have no live callers, and the
`commitsAhead` seam `tick()` needs is only a test mock. This epic wires B1
(run.ts assembly), B2 (real commitsAhead), and B3 (self-clone + per-task
worktree) — the final gap before LP-A1 can run.

## Decision Anchors

- Reviewer/session findings on Epic 019.7: `run.ts:100` uses sync `buildRealDeps`;
  `featureDir = join(slot.repo, ".kanthord/features")` with a URL is invalid;
  `commitsAhead` and `verbAdapters` are only injected in tests.
- Existing, unwired building blocks: `git.clone` adapter (`git-local.ts`,
  re-run-safe), `dispatchWorktree` (`src/slots/worktree.ts`, per-task
  worktree+branch, lease-capped, conflict-detecting), `loadIdentity`
  (`git/keyring.ts`), the Epic 019.7 async `buildRealDeps` (verbAdapters + PAT).
- SU1 custody posture (019 proof-run preamble): git over HTTPS authenticates via
  `http.extraHeader` with the per-identity token — the token is never persisted
  into the remote URL.
- Ulrich (2026-07-12): **B3 is a must-have — the system must clone a repo itself**
  (no operator pre-clone).

## Stories

- `001-self-clone-authenticated-checkout.md` - the daemon clones the slot repo URL
  into a local checkout under the data root and configures HTTPS auth
  (`http.extraHeader` with the identity token); idempotent / re-run-safe; the
  token is never written to the persisted remote URL. (B3a)
- `002-per-task-worktree-dispatch.md` - `tick()` runs each dispatched task in its
  own worktree+branch off the local checkout via `dispatchWorktree`, and the
  session (and its commits) happen there. (B3b)
- `003-commits-ahead-seam.md` - a git-backed `commitsAhead(branch, base)` (real
  `git rev-list --count base..branch`) constructed in the live deps and consumed
  by `tick()` delivery. (B2)
- `004-run-assembles-live-path.md` - `run.ts` self-clones → local repo root,
  derives `featureDir`/db from it, loads the identity, calls the **async**
  `buildRealDeps` (identity + identityFile + patternRegistry + repo coords), and
  threads `verbAdapters` + `commitsAhead` + reconcile into `runDaemon`. (B1)

## Verification Gate

- `npm run typecheck` exits 0; `npm test` green — new hermetic tests pass (clone +
  auth against a local bare "remote"; worktree dispatch; commitsAhead on a temp
  repo; run.ts assembly), the full suite shows no regression, the zero-network
  guard stays green (git against local temp remotes only).
- `node src/cli/run.ts --help` exits 0.
- **Live proof (LP-A1):** booting the daemon on the `kanthord-verify` slot clones
  the repo, runs a signed-off feature in a worktree, and delivers a real PR
  through the broker (Epic 019.7 gate) — the end-to-end path this epic completes.

## Dependencies

- Epic 019.7 (broker delivery capability in buildRealDeps/tick).
- Epic 019.6 (provider session), Epic 016 (`dispatchWorktree`, repo slots).

## Non-Goals

- **No feature ingestion / authoring pipeline (B4)** — how a signed-off `tdd@1`
  feature is authored + compiled into the daemon's store is operator-provided for
  LP-A1; not built here.
- **No multi-repo / multi-slot** — single sandbox slot (Phase 2A). Multi-repo is
  Phase 2B (Epic 030).
- **No new broker/worktree mechanism** — assembles `git.clone`,
  `dispatchWorktree`, `loadIdentity`, and the 019.7 async deps.
