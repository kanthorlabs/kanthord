# EPIC 007.9 — E2E resilience: workspace-prep root check + provider transient-retry

> **Authored 2026-07-22** from the post-007.8 TODO-API real-model E2E
> (run `e2e-0079`; full evidence + all findings in
> `007.9-e2e-findings.md`), then hardened through an adversarial debate
> (opencode/plan — see "Debate deltas" at the bottom). The E2E drove the
> 5-endpoint TODO API from an empty repo through the CLI with
> `openai-codex / gpt-5.6-terra`. The pipeline worked end to end (2/5 endpoints
> landed through build → candidate → approve → land → dependency-unblock →
> conflict-detect → retry; partial CRUD program proof passed: POST 201, GET 200,
> unknown 404). Two **major** defects surfaced; the run only stalled at 2/5
> because of a live OpenAI overload outage — which is itself defect #2.

## Goal

After 007.9 the daemon survives the two failure modes that made this real-model
run fragile:

1. **Workspace prep** no longer misreads a checkout directory. A directory that
   is not its **own** git checkout root — the empty, pre-created dir nested
   inside another repo that failed this run — is handled correctly: cloned
   fresh when empty, reused when it is a genuine root-checkout of the expected
   remote, and rejected with an accurate message otherwise. Real git/spawn
   errors and wrong-origin roots stay hard errors (no masking).
2. **A transient provider error no longer aborts a task outright.** kanthord
   retries at the narrowest proven-safe boundary with bounded, jittered
   backoff, and only fails the task after retries exhaust — preserving the full
   attempt history so a persistent outage is never masked as success.

## Verification Gate

Gates: `npm run verify` (typecheck + test + verify:handoff + lint + db status)

Proof: two hermetic, copy-paste-runnable blocks against the built program — no
network, no real model, no credentials. Both use the fake agent runner and a
**local** git remote so they are deterministic.

```sh
# ── Proof A — WORKSPACE-PREP: empty nested checkout dir clones fresh, no
#    "wrong origin" (the exact failing setup from run e2e-0079). ──
set -euo pipefail
D=".data/e2e-079-wpproof"; rm -rf "$D"; mkdir -p "$D"
# a local bare "remote" seeded with one commit
git init -q --bare "$D/remote.git"
git clone -q "$D/remote.git" "$D/seed"
mkdir -p "$D/seed/src"; printf 'export const x=1;\n' > "$D/seed/src/todo.mjs"
git -C "$D/seed" add -A
git -C "$D/seed" -c user.email=e@e -c user.name=e commit -qm init
git -C "$D/seed" push -q origin HEAD:main
# THE TRIGGER: home exists, is empty, and is nested inside the kanthord repo
mkdir -p "$D/home"
export KANTHORD_DB="$PWD/$D/k.db"
node src/main.ts db migrate >/dev/null
PROJECT=$(node src/main.ts create project --name wp-proof | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
REPO=$(node src/main.ts create repository --project "$PROJECT" --name verify \
  --remote-url "file://$PWD/$D/remote.git" --branch main --auth ambient \
  --path "$PWD/$D/home" | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
scripts/e2e/make-landing-graph.sh "$D/graph"   # fake-agent graph (no model)
INIT=$(node src/main.ts import graph "$D/graph" --create --project "$PROJECT" \
  --bind source=$REPO --bind provider=$REPO --bind cred=$REPO \
  >/dev/null; node -e "console.log(require('fs').readFileSync('$D/graph/.kanthord-export.json','utf8'))" | grep -oE '"initiativeId":"01[0-9A-HJKMNP-TV-Z]{24}"' | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
node src/main.ts run daemon --until-idle --poll-interval 200
# MUST show the root task at awaiting_confirmation (built), NOT "failed":
scripts/e2e/e2e-status.sh "$INIT"
# ASSERT (must print PASS):
node src/main.ts get task --id "$(scripts/e2e/e2e-status.sh "$INIT" | awk 'NR==2{print}' >/dev/null; node src/main.ts list task --initiative "$INIT" --json | node -e "let s=\"\";process.stdin.on(\"data\",d=>s+=d).on(\"end\",()=>{const t=JSON.parse(s).find(t=>t.dependencies.length===0);process.stdout.write(t.id)})")" --json \
  | node -e "let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{const t=JSON.parse(s);process.exit(t.status!=='failed'?0:1)})" \
  && echo "PASS: workspace prep cloned into the empty nested dir; no wrong-origin failure" \
  || { echo "FAIL: root task failed"; exit 1; }
```

```sh
# ── Proof B — PROVIDER-RESILIENCE: a task whose provider call fails
#    transiently K times still lands; the failures are retried, not fatal. ──
# 007.9 adds a fake-runner injection: --fail-transient <taskId>:<count> makes
# run() return a transient-classified failure the first <count> times, then
# succeed. (Injection lives in the fake runner; WHERE the real retry loop sits
# is decided by Story B's spike, but the observable contract below is fixed.)
set -euo pipefail
D=".data/e2e-079-prproof"; rm -rf "$D"; mkdir -p "$D"
git init -q --bare "$D/remote.git"; git clone -q "$D/remote.git" "$D/seed"
mkdir -p "$D/seed/src"; printf 'export const x=1;\n' > "$D/seed/src/todo.mjs"
git -C "$D/seed" add -A && git -C "$D/seed" -c user.email=e@e -c user.name=e commit -qm init
git -C "$D/seed" push -q origin HEAD:main
export KANTHORD_DB="$PWD/$D/k.db"; node src/main.ts db migrate >/dev/null
PROJECT=$(node src/main.ts create project --name pr-proof | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
REPO=$(node src/main.ts create repository --project "$PROJECT" --name verify \
  --remote-url "file://$PWD/$D/remote.git" --branch main --auth ambient \
  --path "$PWD/$D/home" | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
scripts/e2e/make-landing-graph.sh "$D/graph"
node src/main.ts import graph "$D/graph" --create --project "$PROJECT" \
  --bind source=$REPO --bind provider=$REPO --bind cred=$REPO >/dev/null
INIT=$(grep -oE '"initiativeId":"01[0-9A-HJKMNP-TV-Z]{24}"' "$D/graph/.kanthord-export.json" | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
ROOT=$(node src/main.ts list task --initiative "$INIT" --json | node -e "let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{process.stdout.write(JSON.parse(s).find(t=>t.dependencies.length===0).id)})")
# fail the root task's provider call twice, then it must succeed (maxRetries >= 2):
node src/main.ts run daemon --until-idle --poll-interval 200 --fail-transient "$ROOT:2"
node src/main.ts get task --id "$ROOT" --json \
  | node -e "let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{process.exit(JSON.parse(s).status==='awaiting_confirmation'?0:1)})" \
  && echo "PASS: task survived 2 transient provider errors and reached awaiting_confirmation" \
  || { echo "FAIL: transient errors were not retried"; exit 1; }
# and the retry attempts are recorded (not silently swallowed):
node src/main.ts list event --after 0 --limit 1000 --json | grep -c "provider.retry" \
  | node -e "let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{process.exit(Number(s.trim())>=2?0:1)})" \
  && echo "PASS: >=2 provider.retry events recorded" || { echo "FAIL: retries not observable"; exit 1; }
```

## Stories

- **A — Workspace-prep: inspect the checkout root, not "inside a repo".**
  Replace the boolean `isGitRepo()` in `src/workspace/local.ts` with one
  inspection function returning a discriminated state — `root-checkout` |
  `enclosing-checkout` | `bare` | `not-a-repo` | `git-error` — by comparing
  `realpath(git rev-parse --show-toplevel)` to `realpath(homePath)`. Prep then:
  clones fresh **only** on `not-a-repo` + confirmed-empty dir; reuses on
  `root-checkout` with the expected origin; errors clearly on wrong-origin root,
  `enclosing-checkout`, `bare`, non-empty non-root; and **re-throws**
  `git-error` (spawn/permission/bad-config/injected `GIT_DIR`) instead of
  clone-falling-through. Use `lstat` (not `access`) so a broken symlink is not
  read as "absent". State the symlink policy explicitly. "Empty" = zero
  directory entries including hidden files. Clone into the confirmed-empty dir
  **under the existing per-repo lock** (no rmdir-then-rename race); add a test
  for the empty-target `rename` behavior rather than assuming it fails.
- **B — Provider transient-retry (spike-first).** Task B1 is a **spike**:
  inspect the pi/codex adapter + `pi-session.ts` to establish (i) what Codex /
  its SDK already retries and (ii) whether the adapter owns/can resume the
  session — so we do not stack a redundant retry layer. Then implement: an error
  **classifier** in the provider adapter that raises a typed
  `TransientProviderError` (documented 5xx / overloaded / 429 / clearly
  pre-acceptance transport failures — conservative; "processing error" only when
  provably pre-acceptance) vs fail-fast on auth/invalid-request; and a **retry
  policy** at the narrowest proven-safe boundary with full jitter, `Retry-After`
  support, a max-attempt **and** max-elapsed-time bound, emitting a
  `provider.retry` event per attempt and preserving every attempt's cause on
  exhaustion. Post-acceptance failures do **not** silently retry in place — they
  requeue the whole task from a reset workspace and disclose that external side
  effects are not rolled back. Extend `FakeRunner` with the
  `--fail-transient <id>:<count>` injection used by Proof B.
- **C — Minor ergonomics bundle** (papercuts from the run; small, optional if
  time-boxed): add `list credential|ai-provider|repository --project <id>`;
  make `create` output one consistent `<kind> created: <id>` line; enrich
  `get conflict` to show the target-vs-candidate hunks (not just the candidate's
  file copy); and make `scripts/e2e/e2e-smoke-todo.sh` `realpath` its argument
  so its own documented `/tmp/todo.mjs` invocation works despite the macOS
  `/tmp`→`/private/tmp` symlink (which otherwise breaks the run-guard).

## Non-goals

- No automatic mid-session resume of a Codex turn loop unless Story B1's spike
  proves the adapter supports it safely — otherwise post-acceptance failures
  requeue the whole task (existing `retry`) rather than resume.
- No SSH-vs-HTTPS remote-URL equivalence for origin matching (real gap, but
  orthogonal to these two bugs — deferred to a later ergonomics pass).
- No change to the conflict-recovery loop itself (shipped in 007.8) beyond the
  Story C `get conflict` display polish.
- Not a rewrite of the agent-runner or landing pipeline; extend the running
  program.

## Debate deltas (opencode/plan, 2026-07-22)

The first-draft fix designs were sharpened by an adversarial debate; the
material corrections now baked into the stories above:

- WORKSPACE-PREP: use a **structured inspection state**, not a stronger boolean
  (so `git-error` / bare / wrong-origin are never masked as "clone here"); use
  `lstat`; treat hidden files as non-empty; avoid the rmdir-then-rename race by
  cloning into the confirmed-empty dir under the lock; **verify** (don't assume)
  the empty-target rename behavior.
- PROVIDER-RESILIENCE: made **spike-first** — do not assume the adapter/runner
  boundary or that the workspace is discarded on failure; check existing SDK
  retry first; separate request-retry vs session-resume vs whole-task-retry and
  retry only at the proven-safe boundary; classify conservatively; add jitter /
  `Retry-After` / max-elapsed bound; preserve full failure history.
