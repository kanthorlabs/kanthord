# 019.11 walkFeature Tolerates an Absent Feature Dir

## Outcome

`walkFeature` returns an empty walk when the feature dir does not exist, so the
daemon **idles instead of crashing** on a freshly-cloned repo that has no authored
features yet. Found by the LP-A1 container smoke: the daemon self-cloned
`kanthord-verify` (which has no `.kanthord/features`), then `run-loop` →
`bootDaemon` → `walkFeature(featureDir)` did `readdir` on the absent dir →
`ENOENT: scandir …/features` → fatal exit. This honors the Epic 019.9 Story 001
AC ("an empty or absent featureDir → boot succeeds, no features to dispatch").

## Decision Anchors

- `grammar.ts:108` `readdir(dir)` throws ENOENT on an absent dir; an **empty** dir
  already returns empty groups (`harness.ts:73`) — absent should match empty.
- Caller chain: `run-loop.ts:279 bootDaemon` → `boot.ts:77 walkFeature`.
- Epic 019.9 Story 001 AC: absent featureDir is a no-op, not an error.

## Stories

- `001-walkfeature-enoent-empty.md` - `walkFeature` on a non-existent dir resolves
  to an empty walk (same shape as an empty dir); an existing-but-unreadable dir or
  a malformed story dir still errors as today.

## Verification Gate

- `npm run typecheck` exits 0; `npm test` green — absent-dir test + existing
  walkFeature/empty-dir tests pass; no regression; guard green.
- LP-A1 container smoke: the daemon boots on the cloned `kanthord-verify` (no
  features) and idles instead of crashing.

## Dependencies

- Epic 019.8 (self-clone), Epic 019.9 (compile-on-boot).

## Non-Goals

- No change to malformed-story-dir error behavior; no change to empty-dir behavior.
