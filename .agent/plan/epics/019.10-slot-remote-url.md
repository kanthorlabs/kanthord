# 019.10 Slot Accepts a Remote URL Repo

## Outcome

`loadRepoSlot` accepts a slot whose `repo` is a **remote URL** (the daemon
self-clones it per Epic 019.8), instead of rejecting everything that is not
already a local git work tree. Found by the LP-A1 container smoke: `run.ts` →
`loadRepoSlot` ran `git rev-parse --is-inside-work-tree` with the URL as cwd →
`spawn ENOENT` → "path is not inside a git work tree", so the daemon could never
boot on a URL slot. Local-path slots keep their existing validation.

## Decision Anchors

- `repo-slot.ts:132-149` validates `repo` as a local git work tree; Epic 019.8
  `bootstrapLocalCheckout` self-clones a URL `repo` — the two are incompatible.
- The Epic 019 proof-run slot uses `repo:
  https://github.com/kanthorlabs/kanthord-verify.git` (a URL).

## Stories

- `001-loadreposlot-url.md` - `loadRepoSlot` accepts a remote-URL `repo`
  (validates URL shape, skips the local-work-tree check); a local-path `repo`
  keeps the existing git-work-tree validation.

## Verification Gate

- `npm run typecheck` exits 0; `npm test` green — new URL-slot test + existing
  local-path slot tests pass; no regression; guard green.
- LP-A1 container smoke: the daemon boots past slot registration on the
  `kanthord-verify` URL slot and self-clones.

## Dependencies

- Epic 019.8 (self-clone from `slot.repo`).

## Non-Goals

- No change to the local-path slot contract; no new clone logic (019.8 owns it).

## Findings Out

- none
