# EPIC 007.13 — Repository publication (land → remote delivery)

## Goal

An operator can publish a landed repository target to its remote from the CLI, as
a step distinct from landing. `publish repository --repository <id> --branch <b>`
pushes the landed branch to the configured remote using the stored credential
(the same GIT_ASKPASS plumbing as clone), **fast-forward-only**: a diverged
remote makes publish **fail loudly** and report the divergence rather than
force-push. Because a local land and a remote push cannot be one atomic
transaction, publication has its own persisted, queryable state per repository
target (`unpublished` / `published@<remoteOID>` / `diverged`).

## Verification Gate

Gates: `npm run verify` (typecheck + test + verify:handoff + lint + db status)

Proof: hermetic, against a local bare remote with the fake agent. Builds an
initiative (007.12 objective-branch workflow), integrates both objectives onto
the initiative branch in the bare home, publishes that branch to the remote,
asserts the remote ref advanced and `get resource` reports it published, then
proves the divergence guard (a moved remote makes publish fail loudly, no force).

```sh
set -euo pipefail
D=".data/e2e-0713-proof"; rm -rf "$D"; mkdir -p "$D"
git init -q --bare "$D/remote.git"
git clone -q "$D/remote.git" "$D/seed"
mkdir -p "$D/seed/src"; printf 'export const base=1;\n' > "$D/seed/src/todo.mjs"
git -C "$D/seed" add -A
git -C "$D/seed" -c user.email=e@e -c user.name=e commit -qm init
git -C "$D/seed" push -q origin HEAD:main
export KANTHORD_DB="$PWD/$D/k.db"
node src/main.ts db migrate >/dev/null
PROJECT=$(node src/main.ts create project --name pub-proof | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
REPO=$(node src/main.ts create repository --project "$PROJECT" --name verify \
  --remote-url "file://$PWD/$D/remote.git" --branch main --auth ambient \
  --path "$PWD/$D/home" | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
printf 'dummy' > "$D/cred.txt"
CRED=$(node src/main.ts create credential --project "$PROJECT" --name c1 --provider openai-codex --value-file "$D/cred.txt" | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
PROV=$(node src/main.ts create ai-provider --project "$PROJECT" --name p1 --provider openai-codex --model gpt-5.6-terra | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
scripts/e2e/make-initiative-graph.sh "$D/graph" >/dev/null
node src/main.ts import graph "$D/graph" --create --project "$PROJECT" --bind source="$REPO" --bind provider="$PROV" --bind cred="$CRED" >/dev/null
INIT=$(node -e 'console.log(JSON.parse(require("fs").readFileSync(process.argv[1]+"/.kanthord-export.json","utf8")).initiativeId)' "$D/graph")
OBJS=$(node -e 'const g=JSON.parse(require("fs").readFileSync(process.argv[1]+"/.kanthord-export.json","utf8"));process.stdout.write((g.objectiveIds||[]).join(" "))' "$D/graph")
set -- $OBJS; OBJ_A=$1; OBJ_B=$2
export KANTHORD_FAKE_AGENT="$PWD/$D/graph/.fake-agent.json"
HOME_GIT="$PWD/$D/home"; [ -d "$PWD/$D/home/.git" ] && HOME_GIT="$PWD/$D/home/.git"
INITBR="kanthord/init/$INIT"
INITREF="refs/heads/$INITBR"

# ── Build both objectives and integrate them onto the initiative branch in home ──
node src/main.ts run daemon --until-idle --poll-interval 200 >/dev/null
node src/main.ts approve objective --id "$OBJ_A" >/dev/null
node src/main.ts run daemon --until-idle --poll-interval 200 >/dev/null
node src/main.ts approve objective --id "$OBJ_B" >/dev/null
LOCAL_HEAD=$(git --git-dir="$HOME_GIT" rev-parse "$INITREF")

# ── Story A/B — publish pushes the initiative branch to the remote (fast-forward) ──
git --git-dir="$D/remote.git" rev-parse --verify "$INITBR" >/dev/null 2>&1 \
  && { echo "FAIL pre: remote already has the init branch before publish"; exit 1; } \
  || echo "PASS pre: remote has no init branch before publish"
node src/main.ts publish repository --repository "$REPO" --branch "$INITBR" 2>&1 | tee /tmp/pub.out
REMOTE_AFTER=$(git --git-dir="$D/remote.git" rev-parse "$INITBR")
[ "$REMOTE_AFTER" = "$LOCAL_HEAD" ] \
  && echo "PASS A: remote $INITBR advanced to the landed local head ($REMOTE_AFTER)" \
  || { echo "FAIL A: remote not updated (after=$REMOTE_AFTER local=$LOCAL_HEAD)"; exit 1; }

# ── Story C — publish state is queryable and now reports published ──
node src/main.ts get resource --id "$REPO" --json 2>/dev/null | node -e "let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{const r=JSON.parse(s);process.exit(r.publication&&/published/.test(r.publication.state)&&r.publication.remoteOID==='$REMOTE_AFTER'?0:1)})" \
  && echo "PASS C: get resource reports publication{state=published, remoteOID}" \
  || { echo "FAIL C: publication state not queryable/incorrect"; exit 1; }

# ── Story D — divergence guard: a moved remote makes publish fail loudly (no force) ──
git clone -q --branch "$INITBR" "$D/remote.git" "$D/other"
printf 'divergent\n' > "$D/other/src/other.mjs"; git -C "$D/other" add -A
git -C "$D/other" -c user.email=e@e -c user.name=e commit -qm diverge
git -C "$D/other" push -q origin HEAD:"$INITBR"   # remote init branch now ahead of our known tip
REMOTE_DIVERGED=$(git --git-dir="$D/remote.git" rev-parse "$INITBR")
node src/main.ts publish repository --repository "$REPO" --branch "$INITBR" 2>&1 | tee /tmp/pub2.out || true
grep -qiE 'diverg|not a fast-forward|remote moved|rejected|stale' /tmp/pub2.out \
  && echo "PASS D: publish refuses a diverged remote (no force-push)" \
  || { echo "FAIL D: publish did not guard divergence"; cat /tmp/pub2.out; exit 1; }
[ "$(git --git-dir="$D/remote.git" rev-parse "$INITBR")" = "$REMOTE_DIVERGED" ] \
  && echo "PASS D2: remote history was NOT overwritten by publish" || { echo "FAIL D2: remote clobbered"; exit 1; }

echo "007.13 PROOF OK"
```

## Stories

- **A — Publish port + git adapter (fast-forward push).** New `src/publication/`
  capability that pushes `refs/heads/<branch>` from the managed home to the remote
  via the reused GIT_ASKPASS plumbing. Fast-forward-only:
  `--force-with-lease=<ref>:<expectedOID>` as a divergence _guard_ only; a non-ff
  push surfaces as a typed `PublishDivergedError` carrying the remote OID.
- **B — `publish repository` CLI + use case.** Resolves the repo, reads the landed
  local head + last-known remote OID, calls the port, persists the result.
  Human-gated. Id-on-stdout / friendly-on-stderr.
- **C — Persisted, queryable publication state.** Per-repository-target state
  (`unpublished` / `published@<remoteOID>` / `diverged`) in storage, surfaced on
  `get resource` (human + `--json`). A partial publish is distinguishable from
  a completed one.
- **D — Delivery-contract documentation.** Document that task `completed` +
  candidate `landed` means locally landed, and delivery is the explicit `publish`
  step.
- **E — Per-initiative claim exclusion (job queue).** `SqliteJobQueue.claim()`
  skips any initiative that already has an in-flight (`running`) task, in one
  atomic `UPDATE … RETURNING`, so an initiative's tasks are serialized even
  across concurrent daemons — protecting the shared per-initiative clone that
  the publication flow's one-commit-per-objective invariant relies on. Different
  initiatives/projects still run in parallel.

## Non-goals

- No automatic push on approve/land; publication stays an explicit operator step.
- No force-push / remote-history rewrite on the normal path.
- No non-`https-token` push topologies beyond the existing credential plumbing;
  branch-protection / fork / PR-based delivery must fail with a clear message.
- No change to landing itself (007.11).
