# EPIC 007.14 — Deterministic stale-candidate transplant (no-model rebuild)

## Goal

Recovering a stale-base candidate first attempts a **deterministic 3-way
transplant** of the candidate onto the moved base (Git's merge machinery, no
model). If it applies with **zero textual conflicts** and the existing
**verification gate passes** in an isolated workspace, the task returns to
`awaiting_confirmation` with a **new candidate SHA + new base SHA** and
**requires fresh human approval** — the prior approval never carries over. On any
textual conflict or gate failure, fall back to a full model rebuild (today's
behavior). The path taken is recorded as an event.

## Verification Gate

Gates: `npm run verify` (typecheck + test + verify:handoff + lint + db status)

Proof: hermetic, with the fake agent and a local bare remote. Two siblings edit
non-overlapping parts of one file; asserts the second is recovered by transplant
with no model build, still requires approval, and an overlapping edit falls back
to a model rebuild.

```sh
set -euo pipefail
D=".data/e2e-0714-proof"; rm -rf "$D"; mkdir -p "$D"
git init -q --bare "$D/remote.git"; git clone -q "$D/remote.git" "$D/seed"
mkdir -p "$D/seed/src"
printf 'export const A=1;\n// ---\nexport const Z=1;\n' > "$D/seed/src/f.mjs"
git -C "$D/seed" add -A
git -C "$D/seed" -c user.email=e@e -c user.name=e commit -qm init
git -C "$D/seed" push -q origin HEAD:main
export KANTHORD_DB="$PWD/$D/k.db"
node src/main.ts db migrate >/dev/null
PROJECT=$(node src/main.ts create project --name transplant-proof | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
REPO=$(node src/main.ts create repository --project "$PROJECT" --name verify \
  --remote-url "file://$PWD/$D/remote.git" --branch main --auth ambient --path "$PWD/$D/home" | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
printf dummy > "$D/cred.txt"
CRED=$(node src/main.ts create credential --project "$PROJECT" --name c1 --provider openai-codex --value-file "$D/cred.txt" | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
PROV=$(node src/main.ts create ai-provider --project "$PROJECT" --name p1 --provider openai-codex --model gpt-5.6-terra | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
scripts/e2e/make-transplant-graph.sh "$D/graph" >/dev/null   # NEW helper (Story D)
node src/main.ts import graph "$D/graph" --create --project "$PROJECT" --bind source="$REPO" --bind provider="$PROV" --bind cred="$CRED" >/dev/null
INIT=$(node -e 'console.log(JSON.parse(require("fs").readFileSync(process.argv[1]+"/.kanthord-export.json","utf8")).initiativeId)' "$D/graph")
export KANTHORD_FAKE_AGENT="$PWD/$D/graph/.fake-agent.json"

builds () { node src/main.ts list event --after 0 --limit 1000 --json | node -e "let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{process.stdout.write(String(JSON.parse(s).events.filter(e=>e.type==='agent.finished').length))})"; }

node src/main.ts run daemon --until-idle --poll-interval 200 >/dev/null   # builds root + siblings
ROOT=$(node src/main.ts list task --initiative "$INIT" --json | node -e "let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{process.stdout.write(JSON.parse(s).find(t=>t.dependencies.length===0).id)})")
node src/main.ts approve task --id "$ROOT" >/dev/null                     # root lands; siblings now stale
B1=$(builds)

SIB=$(node src/main.ts list task --initiative "$INIT" --json | node -e "let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{const t=JSON.parse(s).find(x=>/non-overlap/i.test(x.title));process.stdout.write(t.id)})")
node src/main.ts retry task --id "$SIB" --refresh 2>&1 | tee /tmp/ref.out
B2=$(builds)
# ── Story A/B — deterministic transplant ran WITHOUT a model build ──
[ "$B2" = "$B1" ] \
  && echo "PASS A: --refresh transplanted with NO new model build (agent.finished unchanged: $B1)" \
  || { echo "FAIL A: a model build ran (before=$B1 after=$B2)"; exit 1; }
node src/main.ts list event --after 0 --limit 1000 --json | node -e "let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{process.exit(JSON.parse(s).events.some(e=>e.type==='candidate.transplanted')?0:1)})" \
  && echo "PASS A2: candidate.transplanted event recorded (auditable path)" || { echo "FAIL A2"; exit 1; }
# ── Story C — transplanted candidate requires FRESH approval (new SHAs, back to awaiting) ──
node src/main.ts get task --id "$SIB" --json | node -e "let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{const t=JSON.parse(s);const c=t.landingCandidate;process.exit(t.status==='awaiting_confirmation'&&c&&c.state==='pending'?0:1)})" \
  && echo "PASS C: transplanted sibling is awaiting_confirmation+pending (must re-approve)" || { echo "FAIL C"; exit 1; }
node src/main.ts approve task --id "$SIB" >/dev/null \
  && echo "PASS C2: fresh approval lands the transplanted candidate" || { echo "FAIL C2"; exit 1; }

# ── Story A — overlapping stale sibling falls back to a MODEL rebuild ──
B3=$(builds)
OVL=$(node src/main.ts list task --initiative "$INIT" --json | node -e "let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{const t=JSON.parse(s).find(x=>/overlap/i.test(x.title)&&!/non-overlap/i.test(x.title));process.stdout.write(t.id)})")
node src/main.ts retry task --id "$OVL" --refresh 2>&1 | tee /tmp/ref2.out
node src/main.ts run daemon --until-idle --poll-interval 200 >/dev/null
B4=$(builds)
[ "$B4" -gt "$B3" ] \
  && echo "PASS A3: overlapping candidate fell back to a model rebuild (agent.finished $B3 -> $B4)" \
  || { echo "FAIL A3: no model fallback on textual conflict"; exit 1; }

echo "007.14 PROOF OK"
```

## Stories

- **A — Deterministic transplant in the recovery use case.** In the `retry`
  use case (not the CLI), for a stale-`pending` candidate: attempt a 3-way
  transplant onto the current base in an isolated workspace. Zero conflicts → run
  the existing gate; green → new candidate commit + `candidate.transplanted`
  event. Any conflict or gate failure → fall back to the model rebuild. No model
  on the deterministic path.
- **B — New candidate identity + `retry task --refresh`.** Persist a new
  candidate SHA + new base SHA (never mutate the old row). Add `--refresh`
  (transplant-first, model-fallback); keep `--rebuild` literal (always model);
  plain `retry` unchanged. Logic in the use case; CLI forwards the flag.
- **C — Approval reinvalidation.** A transplanted candidate returns to
  `awaiting_confirmation` + `candidate:pending` and requires fresh human
  approval; the prior approval never authorizes the transplanted result.
- **D — `make-transplant-graph.sh` + audit wiring.** A fake-agent graph with
  non-overlapping vs overlapping sibling edits; wire `candidate.transplanted`
  into the event feed.

## Non-goals

- No trust-relaxation of the gate; the deterministic path runs the same gate in
  isolation.
- No pre-execution serialization / dependency-ordered sibling builds; this epic
  removes only the model cost, not all re-verification.
- No daemon-automatic transplant-and-land; recovery stays operator-initiated.
