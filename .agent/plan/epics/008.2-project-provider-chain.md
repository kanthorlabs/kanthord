# EPIC 008.2 — Project provider assignment + ordered resolution chain

## Goal

A project can **assign** global AI providers (008.1) in an explicit, **total**
order, giving the project a ranked fallback list. A pure resolver computes the
effective **provider chain**: the project's assigned providers **in rank order**,
then the global **default** appended **iff its id is absent** (first-wins
dedup). A project with **no** assignments resolves to `[default]` alone. The
chain is queryable from the real program (`resolve ai-provider --project`), and
that read path is proven to never leak the folded credential. It also makes
008.1's `remove` **assignment-aware**: removing a provider that projects assign
is handled here via `--cascade` or `--replacement`, transactionally with rank
repair. This epic builds and proves the chain; the daemon consumes it in 008.3.
It does not touch the runner.

## Verification Gate

Gates: `npm run verify` (typecheck + test + verify:handoff + lint + db status)

Proof: hermetic. Exercises the full ordered-assignment contract — explicit
`--rank`, duplicate rejection, `unassign`, default-append, no-dup when the
default is already assigned, default-only for an unassigned project,
assignment-aware removal (`--cascade`; `--replacement` with dedup + rank repair),
logged-out providers excluded from the chain — and proves `resolve --json` never
contains the secret.

```sh
set -euo pipefail
D=".data/e2e-0802-chain"; rm -rf "$D"; mkdir -p "$D"
export KANTHORD_DB="$PWD/$D/k.db"
node src/main.ts db migrate >/dev/null
printf 'sk-SECRET' > "$D/c.txt"
reg(){ node src/main.ts register ai-provider --name "$1" --provider openai-codex \
  --model "$2" --value-file "$D/c.txt" | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}'; }
P1=$(reg alpha gpt-5.6-terra)   # default by first-wins convention (008.1)
P2=$(reg beta  gpt-5.6-sol)
P3=$(reg gamma gpt-5.6-luna)
PRJ=$(node src/main.ts create project --name chain-proof | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
chain(){ node src/main.ts resolve ai-provider --project "$1" --json | node -e "
let s='';process.stdin.on('data',d=>s+=d).on('end',()=>process.stdout.write(JSON.stringify(JSON.parse(s).map(p=>p.id))))"; }

# ── Story A/B/C — rank order + default append: rank0 P3, rank1 P2, default P1 ⇒ [P3,P2,P1] ──
node src/main.ts assign ai-provider --project "$PRJ" --provider "$P2" --rank 1 >/dev/null
node src/main.ts assign ai-provider --project "$PRJ" --provider "$P3" --rank 0 >/dev/null
[ "$(chain "$PRJ")" = "[\"$P3\",\"$P2\",\"$P1\"]" ] \
  && echo "PASS A/B/C: explicit --rank total order, then absent default P1 appended" \
  || { echo "FAIL A/B/C: wrong chain ($(chain "$PRJ"))"; exit 1; }

# ── Story B — a duplicate assignment is rejected (status captured; pipefail-safe) ──
if node src/main.ts assign ai-provider --project "$PRJ" --provider "$P2" > "$D/dup.out" 2>&1; then DUP_RC=0; else DUP_RC=$?; fi
{ [ "$DUP_RC" -ne 0 ] && grep -qiE 'duplicate|already' "$D/dup.out"; } \
  && echo "PASS B-dup: duplicate assignment rejected (non-zero exit + message)" \
  || { echo "FAIL B-dup: duplicate assignment not rejected (rc=$DUP_RC)"; cat "$D/dup.out"; exit 1; }

# ── Story C — default already in the list is NOT duplicated (first-wins) ──
node src/main.ts set-default ai-provider --id "$P3" >/dev/null
[ "$(chain "$PRJ")" = "[\"$P3\",\"$P2\"]" ] \
  && echo "PASS C-dedup: default already assigned is not duplicated" \
  || { echo "FAIL C-dedup: default duplicated ($(chain "$PRJ"))"; exit 1; }

# ── Story B — unassign removes a member ⇒ [P2, default(P3)] ──
node src/main.ts unassign ai-provider --project "$PRJ" --provider "$P3" >/dev/null
[ "$(chain "$PRJ")" = "[\"$P2\",\"$P3\"]" ] \
  && echo "PASS B-unassign: unassign removes the ranked member" \
  || { echo "FAIL B-unassign: wrong chain after unassign ($(chain "$PRJ"))"; exit 1; }

# ── Story D — unassigned project resolves to [default] only ──
PRJ2=$(node src/main.ts create project --name empty-proof | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
[ "$(chain "$PRJ2")" = "[\"$P3\"]" ] \
  && echo "PASS D: unassigned project falls back to default only" \
  || { echo "FAIL D: unassigned project did not fall back to default"; exit 1; }

# ── Story D leak gate — resolve --json never contains the secret ──
node src/main.ts resolve ai-provider --project "$PRJ" --json > "$D/resolve.out" 2>&1
grep -q 'SECRET' "$D/resolve.out" \
  && { echo "FAIL leak: secret leaked via resolve --json"; exit 1; } \
  || echo "PASS leak: resolve --json carries no secret"

# ── Story E — removing an ASSIGNED provider needs --replacement or --cascade ──
# P2 is assigned to PRJ (and is not the default). A bare remove is rejected.
node src/main.ts remove ai-provider --id "$P2" > "$D/rm-assigned.out" 2>&1 || true
grep -qiE 'assigned|replacement|cascade' "$D/rm-assigned.out" \
  && echo "PASS E-guard: removing an assigned provider needs --replacement/--cascade" \
  || { echo "FAIL E-guard: assigned provider removed with no intent flag"; exit 1; }

# --cascade removes the provider AND its assignment rows (rank-compacted).
node src/main.ts remove ai-provider --id "$P2" --cascade > "$D/rm-cascade.out" 2>&1
[ "$(chain "$PRJ")" = "[\"$P3\"]" ] \
  && echo "PASS E-cascade: cascade drops the assignment; chain falls back to default" \
  || { echo "FAIL E-cascade: assignment not cascaded ($(chain "$PRJ"))"; exit 1; }

# --replacement rewrites assignments to another provider and DEDUPS the chain.
PRJ3=$(node src/main.ts create project --name repl-proof | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
node src/main.ts assign ai-provider --project "$PRJ3" --provider "$P1" --rank 0 >/dev/null
node src/main.ts assign ai-provider --project "$PRJ3" --provider "$P3" --rank 1 >/dev/null
# Remove P1, rewriting its PRJ3 slot to P3 — already assigned ⇒ dedup to a single P3.
node src/main.ts remove ai-provider --id "$P1" --replacement "$P3" > "$D/rm-repl.out" 2>&1
[ "$(chain "$PRJ3")" = "[\"$P3\"]" ] \
  && echo "PASS E-replacement: rewrite-to-replacement dedups the chain and compacts ranks" \
  || { echo "FAIL E-replacement: wrong chain after replacement ($(chain "$PRJ3"))"; exit 1; }

# ── Story D — a logged_out provider is excluded from the resolved chain ──
# (After Story E, default=P3 and P1/P2 are removed.) PRJ4 assigns active GX ⇒ [GX,P3].
GX=$(reg gx gpt-5.6-terra)
PRJ4=$(node src/main.ts create project --name loggedout | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
node src/main.ts assign ai-provider --project "$PRJ4" --provider "$GX" >/dev/null
[ "$(chain "$PRJ4")" = "[\"$GX\",\"$P3\"]" ] \
  && echo "PASS D-pre: active assigned provider heads the chain, default appended" \
  || { echo "FAIL D-pre: wrong pre-logout chain ($(chain "$PRJ4"))"; exit 1; }
node src/main.ts logout ai-provider --id "$GX" >/dev/null
[ "$(chain "$PRJ4")" = "[\"$P3\"]" ] \
  && echo "PASS D-loggedout: logged_out GX excluded; chain is exactly the active default [P3]" \
  || { echo "FAIL D-loggedout: wrong post-logout chain ($(chain "$PRJ4"))"; exit 1; }

echo "008.2 PROOF OK"
```

## Stories

- **A — Ordered project→provider assignment store (total order).** New
  `project_ai_providers` table (migration 17): `(projectId, providerId, rank)`,
  unique on `(projectId, providerId)` **and** unique on `(projectId, rank)` so no
  two members share a rank — the order is total, not partial. Extend the registry
  port with `assign(projectId, providerId, rank?)`, `unassign`, and
  `listAssigned(projectId)` returning providers strictly ordered by `rank`.
- **B — `AssignAiProvider` / `UnassignAiProvider` use cases + CLI.** Validate the
  project and provider ids (unknown → typed error) and reject a duplicate
  assignment. Omitted `--rank` appends at the current max rank + 1; an explicit
  `--rank` that collides shifts existing members transactionally so ranks stay
  unique and contiguous. CLI: `assign ai-provider --project --provider [--rank]`,
  `unassign ai-provider --project --provider`.
- **C — Pure provider-chain resolver.** Zero-I/O
  `resolveProviderChain(assignedInRankOrder, defaultProvider)` → assigned
  providers in order, then the default appended **iff** its id is not already
  present. Dedup rule is **first-wins by provider id** (defensive: preserves the
  first occurrence even though the store already forbids assigned duplicates).
  Mirrors the shape of `binding-resolver.ts`.
- **D — `resolve ai-provider` query CLI.** Read-only command that loads a
  project's assigned providers + the global default, **excluding any whose
  credential `state` is `logged_out`** (008.1) so a logged-out account never
  appears in the chain, runs the resolver, and prints the ordered chain (human +
  `--json`). Uses the same `resource-view` redaction so the folded credential
  never reaches output. This active-only input is the exact seam the 008.3 daemon
  and 008.4 failover consume, so the preview and the runtime agree.
- **E — Assignment-aware provider removal (unified flag semantics).** Extends
  008.1's `remove ai-provider` for a provider **assigned to projects**. The two
  flags have **one** meaning each, covering both the default pointer and the
  assignment rows so a single flag is never ambiguous:
  - `--replacement <id>` — repoint **every** reference to `<id>`: the default
    pointer (if the removed provider is the default) **and** all
    `project_ai_providers` rows, **deduplicating** when a project already assigns
    `<id>` and compacting ranks.
  - `--cascade` — **delete** the assignment rows (rank-compacted), reporting the
    affected project count; `--cascade` cannot supply a default target, so if the
    removed provider is **also the default**, `--cascade` is rejected and
    `--replacement` is required.
    A bare remove of an assigned provider is rejected. All of this runs in **one
    transaction** (assignments + default repair together). Removal stays keyed by
    record id.

## Non-goals

- No daemon/runner consumption of the chain yet — this epic only builds and
  proves it, but ships a real queryable CLI slice (`resolve ai-provider`), not a
  dead library (008.3 wires it into execution).
- No runtime failover across the chain (008.4).
- No change to the global registry, default convention, or logout semantics
  themselves (008.1) — this epic only adds the **assignment interaction** to
  `remove`.
- No per-provider health/liveness tracking; the chain is static operator-set
  order, not a health ranking.
