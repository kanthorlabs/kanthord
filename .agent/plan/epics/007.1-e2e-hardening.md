# EPIC 007.1 — E2E hardening (observability, import-context, local landing, resource safety)

> **One coupled bug-fix epic (Ulrich, 2026-07-19).** These are all defects
> surfaced by the same successful TODO-API E2E (`007.1-e2e-findings.md`); Ulrich's
> ruling: _"we are fixing bugs, they must come together."_ So Groups **A**
> (observability + safe export), **C** (import context + local landing), and **D**
> (resource lifecycle + secret/transport safety) ship as ONE epic, not a
> `007.x` family. The findings file stays the debate-hardened INPUT; this file is
> the authored epic. **Group B is deliberately excluded** — its debate-hardened
> direction (independent verification ownership, change-authority, whole-suite
> after write-freeze) IS EPIC 008.1/008.2's executor/lane work; 007.1 does not
> re-own it (see Non-goals).
>
> **Load-bearing principle (Group A debate):** events are the authoritative but
> **sensitive** local journal (never leaves the machine); the ONE shareable
> artifact (`diagnostics export`) carries **debugging info only, nothing
> sensitive** — sanitize at the export boundary, not at write. A typed field
> allowlist limits KEYS not VALUES, so safety = eliminate uncontrolled strings +
> runtime exact-schema validation + treat all human-assigned metadata as private.
>
> **Load-bearing principle (Groups C2/D5 debate):** a same-repo dependency's
> `completed` must mean its accepted change is on the canonical branch the
> dependent clones. Landing + authenticated fetch share ONE per-repo+branch lock
> and ONE durable SHA compare-and-swap protocol — they are designed together
> (this is why C2 and D5 cannot live in separate epics). No auto-rebase, no
> separate integration branch, no branch-basing of dependents.

## Goal

After 007.1, the same TODO-API-style flow that the E2E ran is **operable,
inspectable, chainable, and safe** end to end — the defects that forced raw
SQLite pokes, DB edits, secrets in argv, a hand pre-clone, and a manual
`main`-merge are gone. Concretely, one capability: **run a real multi-task
feature through the CLI without leaving the CLI and without leaking secrets.**
(1) **Observability** — the daemon logs each lifecycle transition to stdout,
`get task --result` surfaces summary/verification/commit/files-changed, and a
new `diagnostics export` produces the SINGLE shareable, structurally-sanitized
safe-facts artifact (the private `events` journal stays local-only and is no
longer throttled at capture). (2) **Import context** — `import graph` binds each
task's resource context by alias→id so an imported graph is immediately
**ready for development**, never a non-runnable shell. (3) **Local landing** —
an accepted same-repo task's branch is **landed to the home canonical branch**
under a per-repo lock with a durable SHA protocol, so a dependent task clones and
sees the prior work (the C2 non-merge bug); the same lock+SHA machinery drives
authenticated fetch/reconcile (D5) and a clean git transport identity (D2). (4)
**Resource safety** — typed `update` commands, create-time model validation,
secret input off argv (`--value-file`/hidden TTY, `--value` removed), and
structural omission of credential values from every export/serialization.

## Verification Gate

Gates: `npm run verify` (typecheck + test + verify:handoff + lint + db status).
Hermetic — no network. The suite proves each fix in isolation with fakes:
observability logging + `get task --result` render against a fake run; the
`diagnostics export` schema is validated with **canary tests** proving prompts /
paths / commands / tool-args / repo / branch / commit text / credential values
never reach the safe artifact; `import graph --bind` resolves aliases→ids and
rejects missing/ambiguous binds against fakes; the landing port
(`RepositoryLanding.land`) and the fetch/reconcile compare-and-swap run against
**real git in temp dirs** (ff, merge, typed conflict, crash-idempotent recovery,
lock contention across processes); typed `update` use cases, `ModelCatalog`
validation, the hidden-TTY / `--value-file` secret reader (raw-mode restore in
`finally`, `--value-timeout`), and the structural credential-omission serializer
each get unit tests; the `agent → executor` field is untouched here (EPIC 008
owns that rename).

Proof: two parts, mirroring EPIC 008.1. **Part A** is the deterministic
correctness gate — pure CLI + real git in temp dirs, NO model — and proves
resource safety (D1/D2/D3/D4/D6), import-context binding (C1), local landing
(C2/D5 via a simulated accepted candidate), and the sanitized export (A). **Part
B** is a guarded live-provider smoke that runs one real task and asserts the
observability surfaces (A1 stdout log lines, A2 `--result`, A7 base_commit) on a
real completion. `set -euo pipefail` makes every step assert; expected failures
use `if ! …; then`.

```bash
set -euo pipefail
export KANTHORD_DB="$(mktemp -d)/kanthord.db"
node src/main.ts db migrate
PROJECT=$(node src/main.ts create project --name demo)

# ===================================================================
# Part A — deterministic: resource safety, import-context, local landing, export
# ===================================================================

# ---------- D4: secret NEVER via --value; --value-file (and `-` = stdin) only ----------
if node src/main.ts create credential --project "$PROJECT" --name k1 \
     --provider anthropic --value sk-plaintext 2>/dev/null; then
  echo "FAIL: --value must be removed (D4 secret-in-argv leak)"; exit 1
fi
SECRET_FILE="$(mktemp)"; printf 'sk-from-file' > "$SECRET_FILE"
CRED=$(node src/main.ts create credential --project "$PROJECT" --name k1 \
        --provider anthropic --value-file "$SECRET_FILE")
printf 'sk-from-stdin' | node src/main.ts create credential --project "$PROJECT" \
        --name k2 --provider anthropic --value-file -
# --value-timeout bounds the wait and FAILS (never hangs) with no input on a pipe:
if node src/main.ts create credential --project "$PROJECT" --name k3 \
     --provider anthropic --value-file - --value-timeout 1s < /dev/null 2>/dev/null; then
  echo "FAIL: empty secret must be rejected"; exit 1
fi

# ---------- D6: credential value is STRUCTURALLY omitted from any serialization ----------
node src/main.ts get resource --id "$CRED" --json > /tmp/cred.json 2>/dev/null || \
  node src/main.ts find resource --project "$PROJECT" --name k1 > /tmp/cred.json
if grep -q 'sk-from-file' /tmp/cred.json; then
  echo "FAIL: credential value leaked into resource serialization (D6)"; exit 1
fi

# ---------- D3: create ai-provider REJECTS an unknown (provider, model) pair ----------
if node src/main.ts create ai-provider --project "$PROJECT" --name bad \
     --provider openai-codex --model no-such-model 2>/tmp/d3.err; then
  echo "FAIL: unknown model must be rejected at create (D3)"; exit 1
fi
grep -qiE 'get models' /tmp/d3.err   # error points the user to the catalog
AIP=$(node src/main.ts create ai-provider --project "$PROJECT" --name gpt \
        --provider openai-codex --model gpt-5.6-terra --effort medium)

# ---------- D1: typed update (model change that needed a DB edit in the E2E) ----------
node src/main.ts update ai-provider --id "$AIP" --model gpt-5.6-sol
node src/main.ts get resource --id "$AIP" --json | grep -q 'gpt-5.6-sol'
if node src/main.ts update ai-provider --id "$AIP" --model no-such-model 2>/dev/null; then
  echo "FAIL: update must re-validate the model (D3 at update)"; exit 1
fi

# ---------- D2: repository transport identity — remote-url + auth union, NO token in URL ----------
HOME_REPO="$(mktemp -d)/home"; git init -q -b main "$HOME_REPO"
git -C "$HOME_REPO" -c user.email=a@b.c -c user.name=t commit -q --allow-empty -m init
if node src/main.ts create repository --project "$PROJECT" --name home \
     --remote-url "https://x-access-token:sk@github.com/o/r.git" --branch main 2>/dev/null; then
  echo "FAIL: userinfo/token embedded in remote-url must be rejected (D2)"; exit 1
fi
REPO=$(node src/main.ts create repository --project "$PROJECT" --name home \
        --remote-url "file://$HOME_REPO" --branch main --auth https-token --credential "$CRED")

# ---------- C1: import graph binds context by alias→id → tasks are RUNNABLE ----------
SRC="$(mktemp -d)/graph"; mkdir -p "$SRC"
cat > "$SRC/todo.md" <<'MD'
---
kind: initiative
ref: todo
name: todo
bindings:
  source: repository
  model: ai_provider
  model-auth: credential
---
MD
mkdir -p "$SRC/api"
cat > "$SRC/api/api.md" <<'MD'
---
kind: objective
ref: api
initiative: todo
name: api
context:
  source: source
  model: model
  model-auth: model-auth
---
MD
cat > "$SRC/api/impl.md" <<'MD'
---
kind: task
ref: impl
objective: api
title: implement TODO API
agent: generic@1
---
# Instructions
Build 5 REST endpoints.
# Acceptance Criteria
- [ ] all endpoints return correct status codes
MD
# missing --bind for a declared alias is a fail-fast, non-interactive error:
if node src/main.ts import graph "$SRC" --create --project "$PROJECT" \
     --bind source="$REPO" --bind model="$AIP" 2>/tmp/c1.err; then
  echo "FAIL: unbound alias 'model-auth' must fail before the graph txn (C1)"; exit 1
fi
grep -qiE 'model-auth' /tmp/c1.err
node src/main.ts import graph "$SRC" --create --project "$PROJECT" \
  --bind source="$REPO" --bind model="$AIP" --bind model-auth="$CRED"
INITIATIVE=$(node src/main.ts find initiative --project "$PROJECT" --name todo)
TASK=$(node src/main.ts list task --initiative "$INITIATIVE" --json | node -e 'let s="";process.stdin.on("data",d=>s+=d).on("end",()=>process.stdout.write(JSON.parse(s)[0].id))')
# A5 + C1: get task --json now shows the resolved context bindings (not undefined):
node src/main.ts get task --id "$TASK" --json > /tmp/task.json
node -e 'const t=require("/tmp/task.json");const c=t.context||{};process.exit(c.repository&&c.ai_provider&&c.credential?0:1)' \
  || { echo "FAIL: imported task has no resolved context — not runnable (C1/A5)"; exit 1; }

# ---------- C2 + D5: local landing to the canonical branch under the lock ----------
# Simulate an accepted candidate branch (Part B proves the full agent run):
WS="$(mktemp -d)/ws"; git clone -q "$HOME_REPO" "$WS"
echo 'export const answer = 42;' > "$WS/answer.mjs"
git -C "$WS" -c user.email=a@b.c -c user.name=t add -A
git -C "$WS" -c user.email=a@b.c -c user.name=t commit -q -m 'task output'
CAND=$(git -C "$WS" rev-parse HEAD)
node src/main.ts repo land --repository "$REPO" --workspace "$WS" \
  --base main --candidate "$CAND" > /tmp/land.json
# home/main now CONTAINS the candidate (the C2 non-merge bug is fixed):
git -C "$HOME_REPO" cat-file -e "$CAND"
git -C "$HOME_REPO" merge-base --is-ancestor "$CAND" main
test -f "$(git -C "$HOME_REPO" rev-parse --show-toplevel)/answer.mjs" || \
  { git -C "$HOME_REPO" show main:answer.mjs >/dev/null; }
# re-landing the same candidate is idempotent (crash-recovery contract, D5 SHA protocol):
node src/main.ts repo land --repository "$REPO" --workspace "$WS" \
  --base main --candidate "$CAND" 2>&1 | grep -qiE 'already landed|up to date|no-op'

# ---------- A: diagnostics export = the ONLY shareable artifact, sanitized ----------
node src/main.ts diagnostics export --initiative "$INITIATIVE" --out /tmp/diag.json
# schema present:
node -e 'const d=require("/tmp/diag.json");process.exit(d.schemaVersion&&Array.isArray(d.records)?0:1)' \
  || { echo "FAIL: export missing safe-facts schema"; exit 1; }
# CANARY: no sensitive strings reach the shareable artifact:
for needle in "$HOME_REPO" "sk-from-file" "gpt-5.6" "answer.mjs" "$CAND" "github.com" "npm test"; do
  if grep -qF "$needle" /tmp/diag.json; then
    echo "FAIL: sensitive value '$needle' leaked into diagnostics export (A canary)"; exit 1
  fi
done
echo "Part A OK"

# ===================================================================
# Part B — guarded live-provider smoke: observability on a real completion
# ===================================================================
if [ -z "${KANTHORD_LIVE:-}" ]; then echo "Part B skipped (set KANTHORD_LIVE=1)"; echo "PROOF OK (A)"; exit 0; fi

# Run the one imported task to completion against the real bound provider.
node src/main.ts daemon run --until-idle > /tmp/daemon.log 2>&1
# A1: the daemon LOGGED lifecycle to stdout (silent-daemon bug fixed):
grep -qiE 'claim' /tmp/daemon.log
grep -qiE 'verif' /tmp/daemon.log
grep -qiE 'complet|await' /tmp/daemon.log
# A2: get task --result surfaces summary + verification + commit + files changed:
node src/main.ts get task --id "$TASK" --result > /tmp/result.txt
grep -qiE 'summary' /tmp/result.txt
grep -qiE 'commit' /tmp/result.txt
# A7: base_commit is recorded (was null in the E2E):
node src/main.ts get task --id "$TASK" --json | \
  node -e 'let s="";process.stdin.on("data",d=>s+=d).on("end",()=>{const r=JSON.parse(s).result||{};process.exit(r.baseCommit?0:1)})' \
  || { echo "FAIL: base_commit still null (A7)"; exit 1; }
echo "PROOF OK (A + B)"
```

## Stories

Bullets = capabilities; a vertical-slice plan reslices them into Story/Task files
under `.agent/plan/stories/007.1-e2e-hardening/`, each carrying its own failing
end-to-end assertion. Grouped by the finding groups; every design ruling is in
the decision-notes sections below.

### Group A — Observability & the single sanitized export

- **A1 — daemon lifecycle logging to stdout.** The `RunDaemon` use case emits
  readable, real-name lifecycle lines (claimed / agent started / verifying +
  result / outcome + reason) to a `Logger` port the CLI wires to stdout. Local
  machine, so full detail is fine (layer 2). No throttling of the underlying
  record.
- **A2/A5 — local inspection CLIs read the private journal directly.** `get task
--result` (or `show task`) prints summary, verification evidence, `commit_sha`,
  and files changed from `task_results`; `get task --json` includes the resolved
  `context` bindings (A5, no longer `undefined`). Local-only, sensitive-ok.
- **A3 — un-throttle the capture; throttle only the display.** `agent.progress`
  is recorded for every occurrence in the `events` journal; the 1-per-5s cap
  moves to the `events` feed DISPLAY only, never the record.
- **A4 — verification is a journal event.** Emit `task.verification`
  (verifierKind, exitClass, durationMs, timedOut) at start/end into the private
  journal; the sanitized export projects only the safe subset (never command
  text or output).
- **A6 — turn/token/cost are recorded** as numeric journal fields (safe to
  export).
- **`diagnostics export` — the ONE shareable artifact (the sanitization
  boundary).** A new observability module projects the private journal into a
  **closed safe-facts schema** (`schemaVersion, sessionRef, taskRef, seq,
timestamp, kind, outcomeCode, reasonCode, toolCategory, exitClass, durationMs,
turns, tokensIn, tokensOut`). Enforcement: closed record kinds + closed error
  enum (unknown → `internal_unclassified`); NO `message` / `metadata` /
  `Record<string,unknown>`; runtime exact-schema validation rejecting unknown
  keys; explicit field-by-field serialization (never spread); opaque RANDOM
  `taskRef`/`sessionRef` (not hashes) with `seq` gap detection; import restriction
  so only this module emits the artifact; a bounded, rotated, owner-only file;
  a preview of record kinds + field names before sharing. **Canary tests** prove
  prompts / paths / commands / tool-args / errors / credentials never appear.

### Group C — Import context binding & local branch landing

- **C1 — bind context on import so tasks are runnable.** Packages declare
  **binding aliases** (`source: repository`, `model: ai_provider`, `model-auth:
credential`); objectives assign context to aliases (package-local lexical
  default); tasks override per-key. `import graph … --bind source=<repo-id>
--bind model=<aip-id> --bind model-auth=<cred-id>` maps aliases → concrete
  resource **ids** (names accepted as CLI shorthand, rejected on 0/multiple
  matches; credentials by id/alias, never value). Before the graph transaction:
  validate provider↔credential compatibility AND each task's effective context
  against its executor's required/forbidden binding set; fail-fast
  non-interactive with one complete error report (`--interactive` optional). The
  graph format version bumps (context now round-trips + applies; absent context
  preserves existing live bindings — inheritance must not silently replace them).
- **C2/D5 — canonical-branch landing + authenticated fetch, ONE lock + SHA
  protocol.** Redefine same-repo `completed`: A `completed` ⇒ A's accepted change
  is on the canonical branch B clones. A narrow **local** port
  `RepositoryLanding.land(candidate) → LandingResult` (synchronous git) lands the
  accepted candidate onto the home canonical branch under a **cross-process
  per-repo+branch lock**; durable candidate metadata (repoId, baseSHA,
  candidateSHA, ref, target, state); ancestry → fast-forward or merge → typed
  conflict (conflict ≠ executor failure; dependents blocked); crash-idempotent
  recovery from SHAs (also populates A7 `base_commit`). Landing is part of
  **approval/completion** (today `approveTask` only promotes the proposal ref in
  the workspace — never lands). `generic@1`: no-change completes; changed →
  accepted-candidate, human approval before landing by default (optional
  `auto-approve`); `tdd@1` already fits. **Every online prepare** acquires the
  same lock, resolves D2 auth fresh, fetches `origin`, classifies ancestry, and
  updates local `main` via compare-and-swap (ff when behind / keep when ahead /
  typed divergence otherwise), then clones the workspace from the **recorded
  canonical SHA** before releasing the lock. Explicit `sync` reconciles
  divergence with a merge commit (clean) or typed human resolution (conflict) —
  NO auto-rebase, NO integration branch. Offline/auth-fail → fail unless an
  explicit cached policy records cached-mode + last-fetched SHA + fetch-time +
  base SHA.
- **Shared integration lifecycle as domain state, not an operational interface.**
  Model `Executor → ChangeCandidate → Acceptance → Integration → LandedChange`
  as domain state/artifacts. Reject `merge@1`-as-executor and an explicit
  integration graph node. Build only the narrow local `RepositoryLanding` port
  now; do NOT force the future `pr@1` (remote/async) to implement it.

### Group D — Resource lifecycle & secret/transport safety

- **D1 — typed update commands** (reject a generic `--attr k=v`). Immutable:
  `id`, `projectId`, `type`, Credential/AIProvider `provider`. Mutable: `name`,
  `branch`, `path`, `remoteUrl`, `auth`, Credential `value`, AIProvider
  `model`/`effort`/`baseUrl`. Optional fields cleared only via explicit
  `--clear-*` flags (omission ≠ clear). A repo update must NOT silently rewrite a
  cached clone's origin (require cache absent or explicit reclone). Late-binding:
  running tasks keep loaded values, queued/retried get current, completed are
  historical. Credential `value` never via `--value` (see D4).
- **D2 — repository transport identity** (no longer "minor"). Ownership is
  **repository-scoped**, not per-task (git auth is transport config; per-task
  collides with the shared home clone — confused deputy). New Repository shape:
  clean `remoteUrl` (rejects embedded userinfo/tokens; supports SSH/self-hosted)
  - `auth` union `{ambient} | {https-token, credentialId} | {ssh-agent}`; migrate
    the hardcoded `org/name` → derived URL away. Secure injection: `GIT_ASKPASS` +
    a static non-secret askpass program, `execFile` (no shell),
    `GIT_TERMINAL_PROMPT=0`, empty `credential.helper`, strip `GIT_TRACE*` /
    `GIT_CURL_VERBOSE` from the child env, token never in URL/argv/config/script,
    redact stderr before any log/event. NEVER token-in-URL or `-c
http.extraHeader`. Cache: resolve the credential fresh per network op; cache
    identity = normalized `remoteUrl`, never the credential; rotation → explicit
    purge/reclone. SSH = clean remote + ambient agent only this epic; HTTPS+token
    is the deterministic default. Migrate the E2E pre-clone hack by audit +
    **prefer reclone** over origin-scrub.
- **D3 — model validation via an app-owned `ModelCatalog` port** (pi adapter) at
  create AND update, with authoritative validation kept at session open (one
  source, no pi in app/domain). Reject unknown `(provider, model)` pairs strictly,
  point to `get models`. Drop the `--allow-unknown-model` / `--base-url` escape
  hatch (baseUrl only applies after the pair resolves). Future-planned config →
  an explicit disabled/unresolved state, not a broken resource.
- **D4 — secret input off argv.** Remove `--value`. Interactive TTY → a tested
  hidden-TTY reader (raw mode, restore in `finally`). Automation → `--value-file
<path>` (`--value-file -` = stdin); sources mutually exclusive; no implicit
  stdin, no `--value-env`. `--value-timeout <duration>` (default 3m) bounds the
  wait; on timeout the CLI FAILS with a clear error + non-zero exit (never
  hangs). Newline contract: read to EOF, strip ONE trailing LF/CRLF, reject
  empty, preserve other bytes. Value never in results/errors/logs/exports.
- **D6 — structural omission of credential values.** Credential values are
  **structurally omitted** from every resource export / error serialization /
  diagnostic (not string-redacted — OAuth JSON nesting defeats exact-string
  redaction). Ties to the Group A export boundary.

## Non-goals

- **Group B (verification scoping / change authority) is NOT here.** Scoped
  single-file testing, independent verification ownership, the whole-suite
  write-freeze comparison, and the declared allowed/forbidden/approval path
  classes are EPIC **008.1/008.2** executor & lane work (008.1 already owns lane
  policy + change authority). 007.1 does not fix the truncate-test incident nor
  add scope enforcement. The onboarding "how to test one file" capability is
  EPIC 008.4.
- **No remote PR / `pr@1`.** C2's remote-git landing is the future `pr@1`
  (resumable async workflow), designed only when its real shape exists. 007.1
  builds the **local** `RepositoryLanding` port only and does NOT force `pr@1` to
  implement it.
- **No `merge@1`-as-executor, no integration graph node.** Integration is domain
  state/artifacts, not an operator interface or graph boilerplate.
- **No `agent → executor` rename.** EPIC 008 owns that migration; 007.1 leaves
  the field/flag/catalog names as they are.
- **No ambient hierarchical resource defaults.** Import context is bound by
  explicit alias→id, never by entity-stored defaults or resource names as ids
  (names aren't unique, delete+recreate changes authority, entity defaults make
  imports non-reproducible).
- **No auto-rebase, no separate integration branch, no branch-basing of
  dependents.** Divergence is reconciled by merge commit or typed human
  resolution; dependents always clone the canonical branch.
- **No managed SSH keys, no dynamic model catalogs.** SSH = clean remote +
  ambient agent only; dynamic provider models are a separate future capability
  with real metadata + catalog refresh.
- **No DB-at-rest / OAuth-at-rest encryption decision.** Key-outside-SQLite
  encryption is a separate decision; 007.1 enforces the same non-disclosure
  (structural omission, no argv, sanitized export) for all credential types but
  does not add at-rest encryption.
- **No OS containment.** Secret non-disclosure here is structural omission +
  input hygiene + export sanitization; process/network sandboxing is EPIC 009.

## Design notes — traceability to the debate-hardened findings

Every ruling below is carried verbatim-in-spirit from
`007.1-e2e-findings.md` (debate-hardened 2026-07-19; Ulrich rulings). This
section exists so the Story/Task expansion never re-derives a settled answer.

- **A (three layers).** (1) private SQLite `events` journal = authoritative,
  local-only, un-throttled at capture; (2) rich local stdout logging + local
  inspection CLIs read it directly; (3) `diagnostics export` = the single
  sanitization boundary, closed safe-facts schema, canary-tested. Sanitize at
  export, not at write. Typed allowlist limits keys not values → eliminate
  uncontrolled strings + runtime schema validation + private-by-default metadata.
- **C1.** No ambient defaults; alias→id binding; ids authoritative, names
  shorthand; validate compatibility + executor binding set before the graph txn;
  fail-fast non-interactive; graph format version bump; absent context preserves
  live bindings.
- **C2/D5 (designed together).** Redefine `completed`; landing is part of
  approval/completion; one cross-process per-repo+branch lock; durable SHA
  compare-and-swap (fixes A7); ff/merge/typed-conflict; crash-idempotent; fetch
  every online prepare; clone from the recorded canonical SHA; explicit `sync`;
  no auto-rebase / no integration branch; integration modeled as domain state;
  narrow local `RepositoryLanding` port only.
- **D1.** Typed updates, immutable vs mutable field sets, `--clear-*` semantics,
  no silent origin rewrite, defined late-binding, value not via `--value`.
- **D2.** Repository-scoped transport identity; clean `remoteUrl` + `auth`
  union; `GIT_ASKPASS` secure injection; never token-in-URL; cache by normalized
  URL not credential; prefer reclone; HTTPS+token default, SSH ambient-only.
- **D3.** `ModelCatalog` port validation at create + update; authoritative check
  at session open; strict unknown-pair rejection; drop escape hatches;
  disabled/unresolved state for future config.
- **D4.** Remove `--value`; hidden-TTY reader or `--value-file` (`-`=stdin),
  mutually exclusive; `--value-timeout` (default 3m) fails, never hangs; newline
  contract; value never serialized.
- **D6.** Structural omission (not string redaction) of credential values from
  all serialization; ties to the A export boundary.

## Sequencing & prerequisites

- **D2 (repository transport shape) lands before D1's repo update and before
  C2/D5** — both consume the new `remoteUrl` + `auth` union. The Repository
  shape migration is a maintainer-flavored change (migration + CLI); pair it
  with the D2 story.
- **The A private-journal / safe-facts schema lands before `diagnostics
export`**, and D6's structural omission is validated by the same canary suite.
- **C1's graph-format version bump** coordinates with EPIC 007's format version;
  do not reuse EPIC 007's number.
- This epic is a **bug-fix epic on a running program** (EPIC 006/007 shipped);
  every slice must keep `npm run verify` green and the Part A Proof runnable.
