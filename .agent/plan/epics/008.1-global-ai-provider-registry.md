# EPIC 008.1 — Global AI provider registry + default convention

## Goal

AI providers become **global** (not project-scoped) and carry their own
credential. A non-interactive `register ai-provider` persists one global
`ai_provider` record holding `name`, provider config (`provider`, `model`,
`baseUrl`, `effort`) **and** its secret. The single-default invariant is:
**zero providers ⇒ no default; one or more providers ⇒ exactly one valid
default**. The **first** provider registered auto-becomes the default
(convention), and an operator can flip the default to any registered provider.
Each record's **id is the stable account identity** — two same-kind accounts
(e.g. two ChatGPT logins) are distinct records — and each carries a credential
**lifecycle state** (`active` / `logged_out`) plus a monotonic
`credentialVersion`, so a later logout is a state change and removal is safe
rather than a fragile delete.
This epic adds the global registry **alongside** the existing project-scoped
path — the daemon and `login` are untouched, so the program stays runnable
(project assignment is 008.2; the daemon/login cutover is 008.3). The stored
secret never appears in any command output.

## Verification Gate

Gates: `npm run verify` (typecheck + test + verify:handoff + lint + db status)

Proof: hermetic, non-interactive. Registers global providers, proves the first
is the sole default, flips the default, proves two same-kind accounts coexist as
distinct records, proves `logout` keeps the record as `logged_out` and `remove`
deletes it while repairing the default (with the `--replacement` guard), and
proves the secret leaks to **no** command's stdout **or** stderr.

```sh
set -euo pipefail
D=".data/e2e-0801-registry"; rm -rf "$D"; mkdir -p "$D"
export KANTHORD_DB="$PWD/$D/k.db"
node src/main.ts db migrate >/dev/null
printf 'sk-alpha-SECRET' > "$D/a.txt"
printf 'sk-beta-SECRET'  > "$D/b.txt"

# ── Story A/B — first registered provider auto-becomes the sole default ──
# Capture full stdout+stderr of registration for the later leak check.
node src/main.ts register ai-provider --name alpha --provider openai-codex \
  --model gpt-5.6-terra --value-file "$D/a.txt" > "$D/reg1.out" 2>&1
P1=$(grep -oE '01[0-9A-HJKMNP-TV-Z]{24}' "$D/reg1.out" | head -1)
node src/main.ts register ai-provider --name beta --provider openai-codex \
  --model gpt-5.6-sol --value-file "$D/b.txt" > "$D/reg2.out" 2>&1
P2=$(grep -oE '01[0-9A-HJKMNP-TV-Z]{24}' "$D/reg2.out" | head -1)
node src/main.ts list ai-provider --json > "$D/list1.out"
node -e "
let s=require('fs').readFileSync('$D/list1.out','utf8');
const a=JSON.parse(s); const def=a.filter(p=>p.isDefault);
const ok = def.length===1 && def[0].id==='$P1' && a.find(p=>p.id==='$P2').isDefault===false;
process.exit(ok?0:1);
" && echo "PASS A/B: first registered provider is the sole default" \
   || { echo "FAIL A/B: default convention or single-default invariant broken"; exit 1; }

# ── Story C — set-default flips to P2, still exactly one default ──
node src/main.ts set-default ai-provider --id "$P2" > "$D/setdef.out" 2>&1
node src/main.ts list ai-provider --json > "$D/list2.out"
node -e "
let a=JSON.parse(require('fs').readFileSync('$D/list2.out','utf8'));
const def=a.filter(p=>p.isDefault);
process.exit(def.length===1 && def[0].id==='$P2' ? 0 : 1);
" && echo "PASS C: set-default flips the default; still exactly one" \
   || { echo "FAIL C: set-default did not enforce a single default"; exit 1; }

# ── Story A/B/C leak gate — no secret in ANY command's stdout or stderr ──
node src/main.ts get ai-provider --id "$P1" --json > "$D/get.out" 2>&1
cat "$D"/reg1.out "$D"/reg2.out "$D"/list1.out "$D"/list2.out "$D"/setdef.out "$D"/get.out > "$D/all.out"
grep -q 'SECRET' "$D/all.out" \
  && { echo "FAIL leak: credential secret appeared in register/list/get/set-default output"; exit 1; } \
  || echo "PASS leak: no credential secret in any command output"

# ── Story D — two same-kind accounts coexist as distinct records ──
node -e "
let a=JSON.parse(require('fs').readFileSync('$D/list2.out','utf8'));
const same=a.filter(p=>p.provider==='openai-codex');
process.exit(same.length===2 && same[0].id!==same[1].id ? 0 : 1);
" && echo "PASS D-coexist: two openai-codex accounts are distinct records" \
   || { echo "FAIL D-coexist: same-kind accounts not independent"; exit 1; }

# A third ACTIVE provider so default-replacement has a valid target.
node src/main.ts register ai-provider --name gamma --provider openai-codex \
  --model gpt-5.6-luna --value-file "$D/a.txt" > "$D/reg3.out" 2>&1
P3=$(grep -oE '01[0-9A-HJKMNP-TV-Z]{24}' "$D/reg3.out" | head -1)

# ── Story D — logout invalidates the credential but keeps the record ──
node src/main.ts logout ai-provider --id "$P1" > "$D/logout.out" 2>&1
node src/main.ts get ai-provider --id "$P1" --json | node -e "
let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{const r=JSON.parse(s);
process.exit(r.state==='logged_out' ? 0 : 1)})" \
  && echo "PASS D-logout: logout flips state to logged_out, record retained" \
  || { echo "FAIL D-logout: logout did not preserve the record as logged_out"; exit 1; }

# ── Story D — removing the default without --replacement is rejected ──
# (P2 is the current default from Story C.) Assert BOTH a non-zero exit and message.
if node src/main.ts remove ai-provider --id "$P2" > "$D/rm-bad.out" 2>&1; then RM_RC=0; else RM_RC=$?; fi
{ [ "$RM_RC" -ne 0 ] && grep -qiE 'replacement|default' "$D/rm-bad.out"; } \
  && echo "PASS D-guard: removing the default without --replacement is rejected (rc=$RM_RC)" \
  || { echo "FAIL D-guard: default removed without a replacement (rc=$RM_RC)"; cat "$D/rm-bad.out"; exit 1; }

# ── Story D — remove default with --replacement repairs the default atomically ──
node src/main.ts remove ai-provider --id "$P2" --replacement "$P3" > "$D/rm.out" 2>&1
node src/main.ts list ai-provider --json | node -e "
let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{const a=JSON.parse(s);
const gone=!a.some(p=>p.id==='$P2'); const def=a.filter(p=>p.isDefault);
process.exit(gone && def.length===1 && def[0].id==='$P3' ? 0 : 1)})" \
  && echo "PASS D-remove: remove deletes the record and repairs the default to the replacement" \
  || { echo "FAIL D-remove: record not removed or default not repaired"; exit 1; }

# ── Story D leak gate — logout/remove output carries no secret ──
cat "$D"/reg3.out "$D"/logout.out "$D"/rm-bad.out "$D"/rm.out > "$D/all-d.out"
grep -q 'SECRET' "$D/all-d.out" \
  && { echo "FAIL leak: secret leaked via logout/remove output"; exit 1; } \
  || echo "PASS leak: no secret in logout/remove output"

echo "008.1 PROOF OK"
```

## Stories

- **A — Global `ai_providers` store + default pointer.** New global table
  (migration 16) keyed by id, columns `name` (globally unique), `provider`,
  `model`, `baseUrl`, `effort`, the folded credential `value`, a credential
  `state` (`active` / `logged_out`), and a monotonic `credentialVersion` integer.
  The id is the **stable account identity** (same-kind accounts differ only by
  id). The default is a **pointer** (a single-row `ai_provider_default` holding
  one `providerId`) so ">1 default" is structurally impossible; the pointer must
  always reference an **`active`** provider (never a `logged_out` one), so
  "≥1 active provider ⇒ a resolvable default" holds. New `AiProviderRegistry`
  port + SQLite adapter: `register`, `list`, `get`, `getDefault`, `setDefault`,
  `logout`, `remove`.
- **B — Register use case + transactional first-wins default.**
  `RegisterAiProvider` inserts a global provider; the insert **and** the
  first-time default-pointer set happen in **one transaction** (`UnitOfWork`) so
  the invariant "≥1 provider ⇒ exactly one valid default" can never observe a
  gap. Registering the name of an **active** record → duplicate-name error;
  registering the name of a **`logged_out`** record **reactivates** it (restores
  the credential, `state`→`active`, bumps `credentialVersion`) instead of
  creating a second account — `name` is the reactivation key, since `register`
  has no `--id`. `list`/`get` derive `isDefault` from the pointer and **exclude**
  the folded secret (reuse the `resource-view` redaction pattern) — no read path
  returns `value`.
- **C — `set-default` use case + CLI.** `SetDefaultAiProvider` repoints the
  default to an existing **`active`** provider id (unknown id → typed error; a
  `logged_out` target is rejected so the default stays resolvable). CLI verbs
  `register ai-provider` (`--name --provider --model [--base-url] [--effort]
--value-file`), `list ai-provider`, `get ai-provider --id`,
  `set-default ai-provider --id`. Id-on-stdout / friendly-on-stderr; secret
  input reuses the existing `credential-input` `--value-file` path.
- **D — Credential lifecycle: logout + remove.** `logout ai-provider --id`
  flips `state` to `logged_out`, bumps `credentialVersion`, and clears/invalidates
  the folded secret **without deleting the record** — idempotent; name + config
  retained; re-`register`/`login` under the same name reactivates it (Story B).
  Logging out the **current default** requires `--replacement <id>` (an `active`
  provider) so the default never points at a `logged_out` record.
  `remove ai-provider --id` deletes the record; removing the **default** while
  other providers remain **requires `--replacement <id>`** (deterministic — never
  silently changes the effective default), with removal + default repair in **one
  transaction**; removing the **last** provider clears the default (0 ⇒ no
  default). Both are keyed by **record id** so acting on one same-kind account
  never touches its siblings, both emit an **audit line to stderr** (record id,
  provider kind, safe name, operation, replacement id when applicable — never the
  secret; these are operator audit lines, not task-scoped `events`-table rows),
  and neither prints the secret. Logout's guarantee is **local invalidation** —
  pi exposes no remote token revoke, so output says so.
  (Assignment-aware removal — replacement/cascade over `project_ai_providers` —
  is 008.2; enforcing the `credentialVersion` CAS against an in-flight OAuth
  refresh is 008.3.)

## Non-goals

- No project→provider assignment or ordering (008.2).
- No daemon/runner rewire and **no `login` change** — `login` still mints a
  project-scoped credential and the run-time path still uses the existing
  project-scoped `ai_provider` + `credential` bindings; both are cut over in
  008.3. This keeps the login→run path unbroken at this epic's boundary.
- No runtime failover (008.4).
- No **assignment-aware** removal (replacement/cascade over project assignments)
  — that rides with 008.2; and no **runtime enforcement** of the
  `credentialVersion` CAS against in-flight OAuth refresh — that is 008.3. This
  epic only defines the state/version fields and the unassigned logout/remove.
- No **custom / non-builtin** provider type — a custom OpenAI-compatible provider
  (custom `baseUrl` + models) is deferred out of the 008.x series pending
  research; `register`/`list` only accept providers in pi's pinned catalog.
- The `credential` resource type is unchanged — it still backs repository
  https-token git auth (`authCredentialId`); only the _AI_ credential folds into
  the global provider.
