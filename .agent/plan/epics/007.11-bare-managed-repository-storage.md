# EPIC 007.11 — Bare managed repository storage

> Reframed 2026-07-23. The first draft of 007.11 fixed the landed-checkout
> inconsistency (F2) by reconciling the home working tree after each land. An
> adversarial debate (opencode/plan) argued — and cited the code — that the
> **stronger, root-cause** fix is to make the managed home **bare**: a repo with
> no working tree cannot develop the HEAD-vs-worktree skew at all, and it matches
> kanthord's already worktree-free landing design. This epic delivers that, and
> is the **prerequisite** for the objective-branch workflow (EPIC 007.14).

## Goal

After 007.11, a repository's managed home is a **bare, kanthord-owned git object
and ref store** — never a developer checkout. Landing and (later) objective
integration operate purely on objects and refs (`fetch` a candidate object,
`merge-tree`/`commit-tree`, CAS `update-ref`); there is no working tree to fall
out of sync, so finding **F2 is structurally impossible** (no `git status` can
report a landed file as deleted, because there is no working copy). Per-task
workspaces continue to clone from the bare home exactly as before. The bare home
is documented as a managed cache: operators inspect landed work via a fresh
checkout or `git show <ref>:<path>`, not by opening the home directory.

## Verification Gate

Gates: `npm run verify` (typecheck + test + verify:handoff + lint + db status)

Proof: hermetic, copy-paste-runnable against a **local bare remote** with the
fake agent — asserts the home is bare, a candidate lands via refs only, and the
old F2 skew cannot occur.

```sh
set -euo pipefail
D=".data/e2e-0711-bare-proof"; rm -rf "$D"; mkdir -p "$D"
git init -q --bare "$D/remote.git"; git clone -q "$D/remote.git" "$D/seed"
mkdir -p "$D/seed/src"; printf 'export const x=1;\n' > "$D/seed/src/f.mjs"
git -C "$D/seed" add -A
git -C "$D/seed" -c user.email=e@e -c user.name=e commit -qm init
git -C "$D/seed" push -q origin HEAD:main
export KANTHORD_DB="$PWD/$D/k.db"
node src/main.ts db migrate >/dev/null
PROJECT=$(node src/main.ts create project --name bare-proof | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
HOME_PATH="$PWD/$D/home"
REPO=$(node src/main.ts create repository --project "$PROJECT" --name verify \
  --remote-url "file://$PWD/$D/remote.git" --branch main --auth ambient \
  --path "$HOME_PATH" | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
printf dummy > "$D/cred.txt"
CRED=$(node src/main.ts create credential --project "$PROJECT" --name c1 --provider openai-codex --value-file "$D/cred.txt" | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
PROV=$(node src/main.ts create ai-provider --project "$PROJECT" --name p1 --provider openai-codex --model gpt-5.6-terra | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
scripts/e2e/make-landing-graph.sh "$D/graph" >/dev/null
node src/main.ts import graph "$D/graph" --create --project "$PROJECT" --bind source="$REPO" --bind provider="$PROV" --bind cred="$CRED" >/dev/null
INIT=$(node -e 'console.log(JSON.parse(require("fs").readFileSync(process.argv[1]+"/.kanthord-export.json","utf8")).initiativeId)' "$D/graph")
export KANTHORD_FAKE_AGENT="$PWD/$D/graph/.fake-agent.json"

# ── Story A — the managed home is BARE (no working tree) ──
HOME_GIT="$HOME_PATH/.git"; [ -d "$HOME_PATH.git" ] && HOME_GIT="$HOME_PATH.git"
IS_BARE=$(git --git-dir="$HOME_GIT" rev-parse --is-bare-repository 2>/dev/null || echo false)
[ "$IS_BARE" = "true" ] && echo "PASS A: managed home is a bare repository" || { echo "FAIL A: home not bare ($IS_BARE)"; exit 1; }
# a bare repo has no working tree / index at the root
test ! -f "$HOME_PATH/src/f.mjs" && echo "PASS A2: no working-tree files at the bare home root" || { echo "FAIL A2"; exit 1; }

# ── Story B/C — a candidate lands via refs only; no F2 skew possible ──
node src/main.ts run daemon --until-idle --poll-interval 200 >/dev/null
ROOT=$(node src/main.ts list task --initiative "$INIT" --json | node -e "let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{process.stdout.write(JSON.parse(s).find(t=>t.dependencies.length===0).id)})")
MAIN_BEFORE=$(git --git-dir="$HOME_GIT" rev-parse refs/heads/main)
node src/main.ts approve task --id "$ROOT" >/dev/null
MAIN_AFTER=$(git --git-dir="$HOME_GIT" rev-parse refs/heads/main)
[ "$MAIN_AFTER" != "$MAIN_BEFORE" ] && echo "PASS B: main ref advanced by land ($MAIN_BEFORE -> $MAIN_AFTER)" || { echo "FAIL B"; exit 1; }
# the landed file is reachable via the object store (how operators read it)
git --git-dir="$HOME_GIT" cat-file -e "refs/heads/main:src/f.mjs" 2>/dev/null \
  && echo "PASS C: landed file readable from the bare store (git show ref:path)" || { echo "FAIL C"; exit 1; }
# there is NO working tree that could disagree with HEAD (F2 impossible)
git --git-dir="$HOME_GIT" status --porcelain 2>/dev/null | grep -q . \
  && { echo "FAIL C2: bare repo reported working-tree status"; exit 1; } \
  || echo "PASS C2: no working-tree status on the bare home (F2 structurally impossible)"

echo "007.11 PROOF OK"
```

## Stories

- **A — Create a bare managed home.** Repository provisioning clones/initialises
  the home as a **bare** repo (`git clone --bare` / mirror-style) with an
  explicit fetch refspec (`+refs/heads/*:refs/remotes/origin/*`), instead of a
  non-bare clone with the integration branch checked out. Record the home as
  bare in storage.
- **B — Make workspace preparation bare-home-aware.** `src/workspace/local.ts`
  currently **rejects** a bare home (`:473-476`) and assumes a non-bare
  `refs/remotes/origin/<branch>` checkout layout (`:526-589`). Update the
  inspection + fetch/CAS path so a bare home is the expected shape; working
  directories still clone **from** the bare home (a bare repo is a valid clone
  source). The per-initiative isolated-clone provisioning (`--no-hardlinks`,
  no `origin`) is defined by EPIC 007.14 — this epic only guarantees the bare
  home is a valid, consistent clone source.
- **C — Object/ref-only landing (drop checkout/worktree merge).** `src/landing/
git.ts` (`:156-220`) and the manual `land repository` path (`src/apps/cli/
repo.ts:35-62`) use checkout / worktree merge operations that assume a working
  tree. Route all landing through the existing object-level path
  (`fetch` candidate → `merge-tree --write-tree` → `commit-tree` →
  CAS `update-ref`), which already keeps refs/HEAD/index/worktree out of it.
  Remove or replace the checkout-based branch of `land repository`.
- **D — Existing-home migration policy.** Existing non-bare managed homes must
  not be **silently** converted (risking loss of local-only refs / operator
  edits). Choose and implement an explicit policy: recreate the managed home as
  bare on next prep (after verifying it is clean / kanthord-owned), or refuse
  with a clear message telling the operator to remove/recreate it. Document that
  the managed home is a kanthord cache, not a developer checkout
  (`README.md` / `AGENTS.md`).

## Non-goals

- No objective-branch workflow (EPIC 007.14) and no remote publication
  (EPIC 007.12) — this epic only changes home **storage** shape and removes
  worktree-based landing.
- No change to the per-task workspace clone model, the candidate/approve gate
  (007.8), or conflict detection beyond routing landing through the object path.
- No new user-facing checkout of the home; operators read landed content via a
  fresh clone or `git show <ref>:<path>`.

## Debate deltas (opencode/plan, 2026-07-23)

- **Bare storage over post-hoc worktree reconcile** — a `reset --hard` after
  land risks discarding operator/tracked changes and preserves a confusing
  "home is also a checkout" model; a bare home makes F2 impossible and matches
  the worktree-free landing design.
- **Bare is a migration, not a flag** — workspace prep rejects bare, assumes an
  origin-tracking layout, and landing uses checkout/worktree ops; Stories B/C
  address each cited site.
- **Explicit existing-home migration** (Story D) — never silently convert a
  managed checkout.
- **Ship this first** as the prerequisite for 007.14, on its own
  independently-verifiable Proof.
