# EPIC 007.11 — Bare managed repository storage

## Goal

A repository's managed home is a **bare, kanthord-owned git object and ref
store** — never a developer checkout. Landing operates purely on objects and refs
(`fetch` candidate → `merge-tree`/`commit-tree` → CAS `update-ref`); no working
tree exists to fall out of sync. Per-task workspaces still clone from the bare
home. The bare home is a managed cache: read landed work via a fresh clone or
`git show <ref>:<path>`.

## Verification Gate

Gates: `npm run verify` (typecheck + test + verify:handoff + lint + db status)

Proof: hermetic, against a local bare remote with the fake agent — home is bare,
a candidate lands via refs only, no working-tree status possible.

```sh
set -euo pipefail
D=".data/e2e-0711-bare-proof"; rm -rf "$D"; mkdir -p "$D"
git init -q --bare "$D/remote.git"; git clone -q "$D/remote.git" "$D/seed"
mkdir -p "$D/seed/src"; printf 'export const x=1;\n' > "$D/seed/src/f.mjs"
git -C "$D/seed" add -A
git -C "$D/seed" -c user.email=e@e -c user.name=e commit -qm init
git -C "$D/seed" push -q origin HEAD:main
export KANTHORD_DB="$PWD/$D/k.db"
node src/main.ts db migrate >/dev/null
PROJECT=$(node src/main.ts create project --name bare-proof | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
HOME_PATH="$PWD/$D/home"
REPO=$(node src/main.ts create repository --project "$PROJECT" --name verify \
  --remote-url "file://$PWD/$D/remote.git" --branch main --auth ambient \
  --path "$HOME_PATH" | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
printf dummy > "$D/cred.txt"
CRED=$(node src/main.ts create credential --project "$PROJECT" --name c1 --provider openai-codex --value-file "$D/cred.txt" | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
PROV=$(node src/main.ts create ai-provider --project "$PROJECT" --name p1 --provider openai-codex --model gpt-5.6-terra | grep -oE '01[0-9A-HJKMNP-TV-Z]{24}')
scripts/e2e/make-landing-graph.sh "$D/graph" >/dev/null
node src/main.ts import graph "$D/graph" --create --project "$PROJECT" --bind source="$REPO" --bind provider="$PROV" --bind cred="$CRED" >/dev/null
INIT=$(node -e 'console.log(JSON.parse(require("fs").readFileSync(process.argv[1]+"/.kanthord-export.json","utf8")).initiativeId)' "$D/graph")
export KANTHORD_FAKE_AGENT="$PWD/$D/graph/.fake-agent.json"

# ── Daemon run — provisions the managed home (prepare → cloneIntoHome) ──
node src/main.ts run daemon --until-idle --poll-interval 200 >/dev/null

# ── Story A — the managed home is BARE (no working tree) ──
# A bare home IS the git dir itself (no .git subdir); clone --bare renames onto $HOME_PATH.
HOME_GIT="$HOME_PATH"
IS_BARE=$(git --git-dir="$HOME_GIT" rev-parse --is-bare-repository 2>/dev/null || echo false)
[ "$IS_BARE" = "true" ] && echo "PASS A: managed home is a bare repository" || { echo "FAIL A: home not bare ($IS_BARE)"; exit 1; }
test ! -f "$HOME_PATH/src/f.mjs" && echo "PASS A2: no working-tree files at the bare home root" || { echo "FAIL A2"; exit 1; }

# ── Story B/C — a candidate lands via refs only; no F2 skew possible ──
ROOT=$(node src/main.ts list task --initiative "$INIT" --json | node -e "let s='';process.stdin.on('data',d=>s+=d).on('end',()=>{process.stdout.write(JSON.parse(s).find(t=>t.dependencies.length===0).id)})")
MAIN_BEFORE=$(git --git-dir="$HOME_GIT" rev-parse refs/heads/main)
node src/main.ts approve task --id "$ROOT" >/dev/null
MAIN_AFTER=$(git --git-dir="$HOME_GIT" rev-parse refs/heads/main)
[ "$MAIN_AFTER" != "$MAIN_BEFORE" ] && echo "PASS B: main ref advanced by land ($MAIN_BEFORE -> $MAIN_AFTER)" || { echo "FAIL B"; exit 1; }
git --git-dir="$HOME_GIT" cat-file -e "refs/heads/main:src/f.mjs" 2>/dev/null \
  && echo "PASS C: landed file readable from the bare store (git show ref:path)" || { echo "FAIL C"; exit 1; }
git --git-dir="$HOME_GIT" status --porcelain 2>/dev/null | grep -q . \
  && { echo "FAIL C2: bare repo reported working-tree status"; exit 1; } \
  || echo "PASS C2: no working-tree status on the bare home (F2 structurally impossible)"

echo "007.11 PROOF OK"
```

## Stories

- **A — Create a bare managed home.** Provision the home as a bare repo
  (`git clone --bare`) with an explicit origin fetch refspec.
- **B — Make workspace preparation bare-home-aware.** Accept a bare home as the
  expected shape; fetch/CAS on the bare layout; per-task workspaces still clone
  from it.
- **C — Object/ref-only landing.** Route the manual `land repository` path onto
  the existing object path (`preview` → `landPreviewed`); remove the
  checkout/worktree `land()`.
- **D — Existing-home migration policy.** Never silently convert a non-bare home:
  recreate-if-clean or refuse with guidance. Document the home as a bare cache.

## Non-goals

- No objective-branch workflow (007.12) and no remote publication (007.13).
- No change to the per-task workspace clone model or the candidate/approve gate
  (007.8) beyond routing landing through the object path.
- No user-facing checkout of the home.
