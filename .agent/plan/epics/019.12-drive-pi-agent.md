# 019.12 Drive the pi Agent

## Outcome

The live daemon actually **runs** the pi Agent: `spawnAgent` delivers the
assembled task **brief** to the Agent and drives its run loop, so the agent makes
real model calls, executes tools (ring-1-gated), edits the worktree, and commits.
Found by the LP-A1 container run: the daemon cloned, compiled, dispatched, and
created the worktree, but the agent sat inert — `run-deps.ts spawnAgent`
constructs `new Agent(agentOpts)` and wires only `abort`/`waitForIdle`/`reset`,
**dropping the `systemPrompt` brief and never calling `agent.prompt(...)`**. So
zero model calls, zero work. The whole hermetic suite passed only because every
test injects an `agentFactory` double that simulates the run — real Agent
execution was never exercised. This epic wires it.

## Decision Anchors

- `pi-agent-core` Agent API (`node_modules/@earendil-works/pi-agent-core/dist/
  agent.d.ts`): `prompt(input: string): Promise<void>` starts + runs a prompt;
  `waitForIdle(): Promise<void>` resolves after the run and `agent_end`
  listeners settle.
- `spawnPiSession` already assembles + passes `systemPrompt` to
  `piSurface.spawnAgent({ systemPrompt, ... })` (`pi-session.ts:278`); the live
  `spawnAgent` (`run-deps.ts:199`) ignores it and never runs the agent.
- `[[pi-sessions-in-process]]` — sessions are in-process `Agent` objects, driven
  by `agent.prompt` / `waitForIdle`, not subprocesses.

## Stories

- `001-spawnagent-drives-run.md` - `spawnAgent` extracts the `systemPrompt` and
  drives the Agent's run with it (`agent.prompt(brief)`); `waitForIdle` resolves
  only after that run completes. The injectable agent seam is extended so the run
  is observable in tests (a double records the brief it was driven with).

## Verification Gate

- `npm run typecheck` exits 0; `npm test` green — a hermetic test asserts
  `spawnAgent` drives the injected agent with the `systemPrompt` and awaits the
  run; existing spawnAgent/`agentFactory` tests still pass; guard green.
- `node src/cli/run.ts --help` exits 0.
- **Live proof (LP-A1):** the in-container run now records model calls in
  `model_call_log`, the agent writes `slugify.mjs` + test in the worktree and
  commits, and the broker delivers a real PR on `kanthord-verify`.

## Dependencies

- Epic 019.6 (spawnAgent + model/streamFn), Epic 019.8/019.9 (clone/compile/
  dispatch), Epic 019.1 (pi tool surface).

## Non-Goals

- **No change to the tool surface or ring-1 gating** (Epic 019.1/015 own those).
- **No change to the brief assembly** (`spawnPiSession` already builds it).
- **No multi-turn steering / follow-up** — a single driven run per session.
