# 019.15 Real Agent Tools via pi-coding-agent Factories

## Outcome

The driven pi agent gets **real, working file tools** instead of the no-op stubs:
kanthord builds pi-coding-agent's real tool factories (`createReadTool`,
`createWriteTool`, `createEditTool`, `createGrepTool`, `createFindTool`,
`createLsTool`), bound to the task **worktree** as their cwd, and hands them to
the `pi-agent-core` `Agent` it already runs. So a `write`/`edit` tool call
actually lands a file in the worktree, a `read`/`ls`/`grep`/`find` actually reads
it — and ring-1 still gates every effectful call, with `bash` permanently
excluded. This closes the LP-A1 blocker: the agent writes `slugify.mjs` + test,
commits, and the broker opens a real PR.

This is **Approach C** (reuse the tool factories on our existing Agent), NOT the
full `createAgentSession` SDK. See memory `pi-coding-agent-adoption` and the
reuse rule in `CLAUDE.local.md`.

## Decision Anchors

- **The gap**: `src/agent/pi-agent-adapter.ts:87 makeAgentOpts` builds each tool
  name into a stub `AgentTool` whose `execute` returns empty — pi-agent-core
  calls that stub (`agent-loop.js:420`), so every tool is a no-op. Memory
  `pi-tools-are-stubs`.
- **Verified factory surface (installed 0.80.3, importable from the package
  root):** `create<X>Tool(cwd: string, options?): AgentTool` for
  read/write/edit/grep/find/ls. Default `operations` use node `fs` — the
  factories run standalone, no SDK runtime services. Tool names are exactly
  `read/write/edit/grep/find/ls` (already kanthord's `PI_DEFAULT_ALLOWED_MANIFEST`
  names). `write` args `{ path, content }`, `edit` args `{ path, edits[] }` — both
  resolve `path` via `resolveToCwd(path, cwd)`. A `createBashTool` exists and MUST
  NOT be built.
- **cwd == worktree invariant**: the tool cwd MUST equal the directory ring-1
  uses as its `worktree` root (run-loop.ts passes `sessionWorktreePath ?? featureDir`
  to `makeRing1HookAdapter`). If they differ, a relative model path resolves to a
  different absolute path in the tool vs the policy check.
- `spawnAgent` already receives `worktreePath` (`run-deps.ts` PiSurface /
  `pi-session.ts:283`) but ignores it for tools today.

## Stories

- `001-real-tools-built-for-the-worktree.md` - characterize the factory surface
  with a learning test, then build the six real factory tools (no bash) bound to
  the session worktree cwd and wire them into the live Agent, so a real `write`
  lands a file and `read` reads it back (not a stub no-op). (discover + fix)
- `002-ring1-gates-the-real-tools.md` - ring-1's `beforeToolCall` still blocks an
  out-of-worktree / out-of-scope `write`/`edit` made through the REAL pi tool and
  escalates, an in-scope worktree write passes, and `bash` stays absent. (security)

## Verification Gate

- `npm run typecheck` exits 0; `npm test` green — a test asserts a real `write`
  tool call executed by the Agent lands a file in the worktree cwd and `read`
  returns it; a test asserts the tool set excludes `bash`; a test asserts an
  out-of-scope write through the real tool is blocked + escalates while an
  in-scope write passes; existing agent / ring-1 / run-loop tests pass; guard green.
- **Live proof (LP-A1):** the in-container run writes `slugify.mjs` + its test in
  the worktree, commits, and the broker opens a real PR on `kanthord-verify`;
  `model_call_log` has rows; **0 stub no-ops** (files actually change).

## Dependencies

- pi-coding-agent 0.80.3 installed (done). Epic 019.14 (write-scope accepts
  worktree writes), 019.13 (ring-1 bound to worktree + brief names it), 019.12
  (agent is driven).

## Non-Goals

- **No full-SDK adoption** — no `createAgentSession`, `AuthStorage`,
  `ModelRegistry`, `ResourceLoader`, or `SettingsManager` (Approach A rejected).
  Only the tool factories are reused.
- **No change to `provider-session` or `model_call_log`** — provider auth and the
  observability hook stay as they are.
- **No compaction / retry / session-branching / skills** from the SDK.
- **`bash` stays permanently excluded** (Epic 015 decision) — the bash factory is
  never built.
- **No hand-rolled tools** — if a factory cannot be reused as-is, mirror its logic
  per the `CLAUDE.local.md` reuse rule; do not invent from scratch.
