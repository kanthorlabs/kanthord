# 019.13 Agent Workspace + Model-Call Logging

## Outcome

The driven pi agent writes into its **task worktree** (not the feature dir), so
its edits pass ring-1 and land where the broker pushes them; and every model call
is recorded in `model_call_log`. Found by the LP-A1 container run: the agent ran
(Epic 019.12) but ring-1 was given `worktree: featureDir` (`.kanthord/features`)
instead of the task worktree, and the brief never told the agent its workspace —
so its writes resolved to the wrong root, hit `not-in-allowlist`, and escalated;
meanwhile `model_call_log` stayed empty because `appendModelCallRecord` has no
live caller.

## Decision Anchors

- `run-loop.ts:433` passes `worktree: featureDir` to `makeRing1HookAdapter`;
  `sessionWorktreePath` (the real per-task worktree, Epic 019.8) is in scope right
  there (line ~394-407).
- `ring1/hook-binding.ts:67-71` resolves relative tool `path`/`destination` args
  against its `worktree` option; pi's own file tools have no cwd option (they use
  `process.cwd()`), so the agent must be told the worktree explicitly.
- `metrics/model-call-log.ts:45 appendModelCallRecord` writes a row
  (task_id/account_id/model/usage); it has zero non-test callers. AgentOptions
  exposes `onResponse`/`onPayload` hooks (`pi-agent-core` agent.d.ts) for the
  per-call seam.

## Stories

- `001-agent-writes-in-worktree.md` - ring-1 is bound to the **task worktree**,
  and the assembled brief states the worktree as the agent's working directory, so
  the agent's writes land in the worktree and pass ring-1. (B1)
- `002-record-model-calls.md` - each model call in a live session appends a
  `model_call_log` row (task, account, model, token usage) via the agent
  response/stream seam. (S1)

## Verification Gate

- `npm run typecheck` exits 0; `npm test` green — a test asserts ring-1 is bound
  to the session worktree (not featureDir) and the brief names the worktree; a
  test asserts a completed model call appends a `model_call_log` row; existing
  tests pass; guard green.
- **Live proof (LP-A1):** the in-container run writes `slugify.mjs` + test in the
  worktree, commits, delivers a real PR, and `model_call_log` has rows with cost.

## Dependencies

- Epic 019.12 (agent is driven), 019.8 (per-task worktree), 019.4/019.5
  (`model_call_log` + account records).

## Non-Goals

- **No change to the ring-1 policy engine or the tool surface** (015/019.1 own it)
  — only which worktree it is bound to + the brief.
- **No new metrics schema** — `model_call_log` already exists (Epic 019.5).
