# 019.9 Feature Compile Ingestion

## Outcome

A signed-off `tdd@1` feature reaches a running daemon's store two ways: the daemon
**compiles its feature dir on boot** (self-sufficient — `kanthord run` alone
picks up authored features), and an operator can run **`kanthord compile`**
explicitly (to validate/ingest a feature without booting the daemon). Both wrap
the existing `compile()` (`src/compiler/compile.ts`); today `compile()` is called
only by the test harness, so a real daemon's `plan_generation` table is never
populated and `tick()` has nothing to dispatch. This epic closes the last gap
before LP-A1: authored feature markdown → compiled store rows → dispatchable
tasks.

## Decision Anchors

- `run-loop.ts:325` dispatches from `SELECT DISTINCT feature_id FROM
  plan_generation`; nothing in the live path populates that table.
- `compile(featureDir, store, opts: CompileOptions)` (`src/compiler/compile.ts`)
  populates the store; harness usage `compile(dir, store, { repoRegistry })`.
- `bootstrap-live-run.ts` already opens the store (`openStore` + `initSchema`) at
  `<checkout>/.kanthord/db.sqlite` and derives `featureDir` from the checkout —
  the natural place to compile on boot.
- Ulrich (2026-07-12): **both** compile-on-boot **and** a `kanthord compile` CLI.

## Stories

- `001-compile-on-boot.md` - the daemon compiles its feature dir into the store at
  startup (before the first dispatch), idempotently (re-boot does not duplicate or
  error), so `kanthord run` alone ingests authored features.
- `002-kanthord-compile-cli.md` - a `kanthord compile` CLI that opens the store
  for a slot/checkout and runs `compile()`, reporting compiled feature/task counts
  and exiting non-zero on a compile/lint error — operator ingestion + validation
  without booting the daemon.

## Verification Gate

- `npm run typecheck` exits 0; `npm test` green — compile-on-boot test (feature dir
  → dispatchable tasks after boot; re-boot idempotent) and the CLI test pass; full
  suite no regression; zero-network guard green.
- `node src/cli/compile.ts --help` exits 0; `node src/cli/run.ts --help` exits 0.
- **Live proof (LP-A1):** an authored `tdd@1` feature on the `kanthord-verify`
  checkout is compiled (on boot or via `kanthord compile`) and the daemon
  dispatches its tasks — the input to the broker-delivery path (Epics 019.7/019.8).

## Dependencies

- Epic 019.8 (`bootstrapLiveRun` opens the store + derives featureDir).
- Epic 002 (`compile()` — the compiler this epic wraps).

## Non-Goals

- **No new compiler/lint logic** — wraps `compile()` unchanged.
- **No feature-authoring UI / sign-off workflow** — the feature markdown is
  operator-authored; "sign-off" beyond dispatchability is out of scope (dispatch
  is DAG-based, Epic 004).
