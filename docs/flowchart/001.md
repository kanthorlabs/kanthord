# EPIC 001 — Development environment · what exists after this epic

Three views of the finished epic: the **static architecture** (files and the
import-direction rules, machine-enforced by lint), the **runtime flow** of the
Proof command (`node src/main.ts status`), and the **verification bundle**
(`npm run verify`).

## 1. Static architecture — the walking skeleton

Every arrow is an allowed import direction; anything not drawn is forbidden and
fails `npm run lint`.

```mermaid
flowchart TB
    subgraph composition["Composition root"]
        MAIN["src/main.ts<br/><i>the ONLY file that imports concrete adapters</i>"]
    end

    subgraph apps["Driving adapter (thin)"]
        CLI["src/apps/cli/<br/>parse argv · map <code>status</code> → use case · format output"]
    end

    subgraph core["Application core"]
        UC["src/app/status/get-status.ts<br/><code>GetStatus.execute()</code>"]
        PORT["src/storage/port.ts<br/><code>StatusStore</code> port<br/>path · schemaVersion() · journalMode() · taskCount() · close()"]
    end

    subgraph domain["Domain (pure, zero I/O)"]
        GREET["src/domain/greeting.ts<br/><i>hello module — proves the toolchain</i>"]
    end

    subgraph adapter["Driven adapter"]
        STORE["src/storage/sqlite/sqlite-status-store.ts<br/><code>SqliteStatusStore</code>"]
        MIGRATE["src/storage/sqlite/migrate.ts<br/><code>migrate(db, migrations)</code> — the runner"]
        REGISTRY["src/storage/sqlite/migrations.ts<br/><code>MIGRATIONS</code> registry<br/>v1: create tasks table"]
    end

    DB[("SQLite file<br/><code>.data/kanthord.db</code><br/>WAL · user_version=1 · tasks(id)")]

    MAIN -->|wires| CLI
    MAIN -->|constructs| STORE
    MAIN -->|injects| REGISTRY
    CLI -->|calls| UC
    UC -->|"import type"| PORT
    STORE -->|implements| PORT
    STORE -->|runs at open| MIGRATE
    MIGRATE -->|applies| REGISTRY
    STORE --> DB
```

Import rules the lint enforces (Story 2):

- `src/domain/**` imports only `src/domain/**` + `node:*`.
- `src/app/**` imports only `src/domain/**` + `*/port.ts` (`import type`).
- Only `src/main.ts` imports concrete adapters.
- `src/apps/**` never imports adapters or `domain/` internals.
- Tests are carved out: `*.test.ts` may import `node:test`, `node:assert`, and
  (co-located adapter tests) the adapter in their own directory.

## 2. Runtime flow — `node src/main.ts status` (the Proof)

```mermaid
flowchart TD
    START(["node src/main.ts status"]) --> ENV["read KANTHORD_DB<br/>default: .data/kanthord.db"]
    ENV --> MKDIR["mkdirSync(dirname(dbPath), recursive)<br/><i>clean checkout has no .data/</i>"]
    MKDIR --> OPEN["SqliteStatusStore opens/creates the DB file<br/>PRAGMA journal_mode=WAL"]
    OPEN --> MIG{"migrate(db, MIGRATIONS)<br/>read PRAGMA user_version"}

    MIG -->|"pending: version > current"| APPLY["apply each in order,<br/>one transaction each:<br/>up(db) → bump user_version → COMMIT"]
    APPLY -->|throw| ROLLBACK["ROLLBACK<br/>no half-applied schema"]
    APPLY --> DONE["user_version = latest"]
    MIG -->|"nothing pending"| DONE
    DONE --> EXEC["GetStatus.execute()<br/>reads via the StatusStore port"]

    EXEC --> OUT["CLI prints the locked contract:<br/>db: .data/kanthord.db<br/>schema: 1<br/>journal_mode: wal<br/>tasks: 0"]
    OUT --> CLOSE["store.close()"]
    CLOSE --> EXIT(["exit 0<br/><i>stderr may carry the node:sqlite<br/>ExperimentalWarning — accepted</i>"])
```

The migration runner is the piece later epics reuse: **EPIC 002/003 never write
runner code** — they append `{ version: N, name, up }` entries to `MIGRATIONS`
and the same bootstrap applies them idempotently on next start.

## 3. Verification bundle — `npm run verify`

```mermaid
flowchart LR
    V(["npm run verify"]) --> TC["typecheck<br/>tsc --noEmit"]
    TC --> T["test<br/>node --test<br/><i>domain + use case (fakes) +<br/>runner + adapter (temp DB)</i>"]
    T --> H["verify:handoff<br/>VERIFY: PASS"]
    H --> L["lint<br/>eslint .<br/><i>boundary rules + negative proof</i>"]
    L --> P["Proof<br/>node src/main.ts status"]
    P --> OK(["exit 0<br/>repo is healthy"])

    TC -.->|fail| STOP(["non-zero exit"])
    T -.->|fail| STOP
    H -.->|fail| STOP
    L -.->|fail| STOP
    P -.->|fail| STOP
```

  just configured.
